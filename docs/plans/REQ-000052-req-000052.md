## 1. Summary

This plan implements the two-pronged fix specified in the TDD:

1. **Artifact-as-proof fallback** in the Phase C verifier (primary fix) — a substantive
   audit-log presence-miss is no longer fatal if the request's `phase-result-*.json
   .artifacts[]` (or a small set of well-known on-disk paths) contains a freshness-validated,
   ecosystem-matched, content-validated test/build artifact.
2. **Audit-log capture hardening** — a new PostToolUse companion hook backfills Bash
   invocations the existing PreToolUse hook missed, and the PreToolUse hook gains a
   `flock`-wrapped append + `tool_use_id` capture.

All changes are additive and gated by `VERIFICATION_ARTIFACT_FALLBACK` (default `1`).
Anti-fabrication red-team fixtures must continue to refuse.

  - `VERIFICATION_ARTIFACT_FRESHNESS_REQUIRED=0` → returns 0 regardless of mtime.

### TASK-005 — Implement the seven `_check_artifact_*` validators

- **Description.** Implement each of the seven content validators listed in TDD §3.2,
  §3.3. Each validator:
  - Confirms the file is readable and ≤ `VERIFICATION_ARTIFACT_MAX_SIZE_MB`.
  - Runs the validator-specific content check (XML root, JSON shape, regex against
    tail).
  - On success, prints zero or more `k=v` extras lines to stdout (e.g.
    `tests=208`, `failures=0`) and exits 0; on failure, prints nothing and exits 1.
  - Wraps long-running shell-outs (`jq`, `xmllint`/`grep`) in a 5s `timeout` (TDD §6 row 4).
  - The seven validators:
    1. `_check_artifact_junit` (XML `<testsuite[s]>` root, ≥1 `<testcase>`, positive
       `tests` attr).
    2. `_check_artifact_lcov` (≥1 `SF:` and ≥1 `DA:`).
    3. `_check_artifact_istanbul_json` (top-level object whose values have `path` +
       `statementMap`).
    4. `_check_artifact_cobertura_xml` (XML `<coverage>` with `lines-covered` or
       `line-rate`).
    5. `_check_artifact_vitest_json` (top-level `numTotalTests ≥ 1`,
       `numTotalTestSuites ≥ 1`).
    6. `_check_artifact_jest_json` (same shape as Vitest).
    7. `_check_artifact_executor_tail` — tail (last 200 lines) matches any regex in TDD
       §3.3.
- **Files (modify).**
  - `plugins/autonomous-dev/lib/verification/artifact-proof.sh`
- **Dependencies.** TASK-001.
- **Acceptance criteria.**
  - Each validator returns 0 on a positive fixture and 1 on a corrupt/empty/wrong-shape
    fixture (covered by TASK-011 cases).
  - Size cap respected: a 6 MB file with `VERIFICATION_ARTIFACT_MAX_SIZE_MB=5` → 1.
  - 5s timeout on parser shell-out: a maliciously huge XML still terminates and returns 1.
- **Lint/test commands.**
  - `bash -n plugins/autonomous-dev/lib/verification/artifact-proof.sh`
  - `shellcheck plugins/autonomous-dev/lib/verification/artifact-proof.sh`
- **Estimated effort.** **3.5 h** (seven validators × ~30 min each, regex set
  authored from TDD §3.3 verbatim).
- **Track.** A.
- **Risks.**
  - *Medium:* XML parsing without `xmllint` available on every distro. Mitigation:
    accept both `xmllint` and a regex fallback (TDD §3.2 explicitly bounds the
    invariant — XML root + ≥1 `<testcase>`). Regex over the head of the file is
    sufficient.
  - *Low:* `jq` absent on the host. Mitigation: `jq` is already required by the wider
    verifier (it parses `phase-result-*.json`), so its presence is a stable assumption.

### TASK-006 — Implement `verification_artifact_proof_for`

- **Description.** Compose TASK-002..TASK-005. For a given `(REQ_DIR, CLAIM_JSON,
  PHASE)`:
  1. Extract the claim's first-token command.
  2. Compute the candidate path set (TASK-002).
  3. For each candidate, classify kind (TASK-003), check ecosystem match (TASK-003),
     check freshness (TASK-004), run the validator (TASK-005).
  4. On the first validator that returns 0, emit the proof descriptor
     `<kind>:<abs_path>|<extras>` per TDD §4.1 and return 0.
  5. If no candidate passes, return 1 with empty stdout.
  Per-claim wall-clock budget 5 s (TDD §6 row 4) — implemented as an outer `timeout` or
  a deadline loop.
- **Files (modify).**
  - `plugins/autonomous-dev/lib/verification/artifact-proof.sh`
- **Dependencies.** TASK-002, TASK-003, TASK-004, TASK-005.
- **Acceptance criteria.**
  - Positive: JUnit XML in `artifacts[]` matching a `bun test` claim → proof emitted.
  - Positive: lcov in well-known path matching a `vitest` claim → proof emitted.
  - Negative (ecosystem): pytest claim + only lcov.info → empty stdout, returns 1.
  - Negative (staleness): mtime < dispatched_at → empty stdout, returns 1.
  - Negative (content): zero-byte `junit.xml` → empty stdout, returns 1.
  - Wall-clock budget: 200 candidate files all 5 MB does not hang > 6 s.
- **Lint/test commands.**
  - `bash -n plugins/autonomous-dev/lib/verification/artifact-proof.sh`
