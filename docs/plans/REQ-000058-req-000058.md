### TASK-001 (T1): Create `daemon-run-tests.sh` — canonical test re-execution producer

- **ID**: TASK-001
- **Title**: Add `lib/verification/daemon-run-tests.sh` with runner detection, JSON-reporter wrapping, and `test-results.json` writer.
- **Description**: Implement the new file per TDD §4.1 (public interface), §4.1.1 (detector), §4.1.2 (reporter wrapping), §4.1.3 (tail-parse fallback), §4.1.4 (timeouts / cwd / env), §4.2 (schema), and §5.1 (idempotency, ownership). Function `daemon_run_tests <req_dir> <project_root> <phase>` MUST write `<req_dir>/test-results.json` at mode 0600 matching TDD §4.2 schema v1, and MUST emit `daemon_test_rerun_started`/`_completed`/`_skipped_no_runner`/`_timeout` events to `<req_dir>/events.jsonl`.
- **Files**:
  - CREATE `plugins/autonomous-dev/lib/verification/daemon-run-tests.sh`
- **Dependencies**: none.
- **Acceptance Criteria**:
  1. `bash -n plugins/autonomous-dev/lib/verification/daemon-run-tests.sh` returns 0.
  2. `shellcheck plugins/autonomous-dev/lib/verification/daemon-run-tests.sh` returns 0 (or matches project conventions if any suppressions are documented).
  3. Sourcing the file exports `daemon_run_tests` (`declare -F daemon_run_tests` succeeds).
  4. Detector returns `runner="vitest"` when project has `scripts.test` invoking vitest, `runner="jest"` when jest, `runner="npm_script"` when arbitrary, `runner="none"` when no `package.json`.
  5. Artefact schema conforms to TDD §4.2: required keys `schema_version=1`, `phase`, `produced_by="daemon_run_tests"`, `produced_at` (ISO-8601 UTC with `Z`), `runner`, `detected_command`, `invoked_command`, `cwd`, `exit_code`, `duration_ms`, `source`, `tests`, `passed`, `failures`, `errors`, `skipped`, `status`, `output_tail_path`.
  6. Written artefact has file mode `0600` (verify via `stat -f '%Lp'` or `stat -c '%a'`).
  7. Working directory during re-execution is `<project_root>` (not `<req_dir>`) — per TDD §4.1.4.
  8. Return codes match TDD §4.1: `0` = artefact written, `1` = no runner (no artefact), `2` = infrastructure error or timeout (artefact written with appropriate `status`).
  9. Kill-switch `VERIFICATION_DAEMON_RERUN=0` is honoured at the top of the function (early return).
  10. Timeout applied via `resolve_timeout_bin` (or graceful fallback if unavailable); default 300 s, override via `VERIFICATION_DAEMON_RERUN_TIMEOUT_S`.
- **Lint/Test Commands**:
  - `bash -n plugins/autonomous-dev/lib/verification/daemon-run-tests.sh`
  - `shellcheck plugins/autonomous-dev/lib/verification/daemon-run-tests.sh` (if available)
  - `bash -c 'source plugins/autonomous-dev/lib/verification/daemon-run-tests.sh && declare -F daemon_run_tests'`
- **Estimated Effort**: 2.5 h base + 0.5 h buffer = **3.0 h** (200 LOC bash + jq, per TDD §11).
- **Track**: A (producer/verifier).
- **Risks**:
  - **MED — Tail-parse regex divergence** from `artifact-proof.sh:_check_artifact_executor_tail`. *Mitigation*: literal-copy the regex family cited in TDD §4.1.3 (lines 534–614); do not re-derive.
  - **LOW — Reporter flag drift** (vitest/jest CLI). *Mitigation*: use the exact flags in TDD §4.1.2 table; do not "improve."
  - **LOW — Multi-workspace repo detection** (TDD §12 Q3). *Mitigation*: pick project root's `package.json` for this REQ; leave `VERIFICATION_DAEMON_RERUN_CWD` env override as a follow-up flagged in operator docs.

## 9. Notes for Executors

- **Idempotency**: `daemon_run_tests` overwrites the artefact on re-invocation; callers must not depend on preservation (TDD §5.1).
- **Concurrency**: Not designed for concurrent invocation within same `req_dir` — supervisor guarantees single agent per `(request_id, phase)`.
- **Ownership**: All artefacts 0600, daemon uid.
- **Kill-switches**: `VERIFICATION_DAEMON_RERUN=0` fully disables the producer; verifier falls back to prior artifact-proof behaviour (no regression vs today).
- **Do not** modify `command-audit.jsonl` capture logic (`bin/spawn-session.sh:253–273`). ADR-058-01 explicitly declines that option.
- **Do not** bypass or reimplement `artifact-proof.sh`. This design *feeds* it (TDD §2.4).
- **Do not** modify F7 detector logic in `bin/lib/self-heal.sh:492–581`. TASK-007's log-line addition is the sole permitted edit to that file.
