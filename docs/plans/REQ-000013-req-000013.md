### TASK-001 — Add `map_state_status_to_intake()` pure status translator

- **Description:** Implement the pure bash `case`-based status translator per TDD §4.1 / §5.1. Maps `state.json` status vocabulary to the `intake.db` CHECK-constrained vocabulary: `queued→queued`, `running|gate→active`, `paused→paused`, `done→done`, `failed→failed`; any other value echoes empty string and returns `1`. No side effects beyond a caller-driven skip. Place near the other helpers in `supervisor-loop.sh` (above `advance_phase`).
- **Files:** `plugins/autonomous-dev/bin/supervisor-loop.sh` (add function).
- **Dependencies:** none.
- **Acceptance Criteria:**
  - Each valid input maps to the exact token in TDD Appendix C (no CHECK-set member omitted).
  - Unmapped input echoes `""` and returns rc `1`.
  - `cancelled` is **not** producible by this function (owned by orphan/cancel paths — TDD §4.1, NG4).
  - Unit tests U1, U2 (TDD §8.1) pass.
- **Lint/Test Commands:**
  - `bash -n plugins/autonomous-dev/bin/supervisor-loop.sh`
  - `shellcheck plugins/autonomous-dev/bin/supervisor-loop.sh` (if available in env)
  - `bats plugins/autonomous-dev/tests/bats/sync_intake_db_row.bats` (U1/U2 subset)
- **Estimated Effort:** 1.0h (incl. test writing).
- **Track:** A.
- **Risks:** Low — pure function, no I/O. Mitigation: table-driven test covering every vocabulary member.

### TASK-002 — Add `sync_intake_db_row()` ledger-mirror helper (all resilience branches)

- **Description:** Implement the best-effort, non-fatal ledger mirror helper per TDD §4.2 / §5.2. Signature: `sync_intake_db_row(request_id, phase, state_status, ts)`. Behavior: guard on `command -v sqlite3` (ERROR + return 0 if absent), guard on `[[ -f $INTAKE_DB ]]` (WARN + return 0 if missing), call `map_state_status_to_intake` (ERROR + return 0 on unmapped/empty), then run a single parameter-bound `UPDATE requests SET current_phase=:phase, status=:status, updated_at=:ts WHERE request_id=:rid;` followed by `SELECT changes();` in one `sqlite3` heredoc. Capture `2>&1`; on non-zero exit log ERROR; on `changes()==0` log WARN; on success log INFO. Single-quote-escape `request_id` and `phase` (matching orphan path `:2779`). **Always returns 0.**
- **Files:** `plugins/autonomous-dev/bin/supervisor-loop.sh` (add function).
- **Dependencies:** TASK-001.
- **Acceptance Criteria:**
  - Happy path updates `(current_phase, status, updated_at)`; `changes()==1`; INFO logged (U3).
  - `running`/`gate` → `active` with no CHECK violation (U4 / FR-D3).
  - Zero-row match → WARN, rc 0, no crash, no INSERT (U5 / FR-C3).
  - Simulated `sqlite3` error (non-DB/read-only file, or failing PATH stub) → ERROR logged, rc 0, no abort (U6 / FR-C1).
  - `sqlite3` absent (PATH stub rc 127) → ERROR, rc 0 (U7 / FR-C2).
  - Every code path emits exactly one terminal log line (no silent return — NFR-6/SM-6).
  - `updated_at` written equals the passed `ts` (FR-A4).
- **Lint/Test Commands:**
  - `bash -n plugins/autonomous-dev/bin/supervisor-loop.sh`
  - `bats plugins/autonomous-dev/tests/bats/sync_intake_db_row.bats`
- **Estimated Effort:** 3.0h (incl. resilience-branch tests, which dominate).
- **Track:** A.
- **Risks:**
  - **Medium — single-quote escaping / heredoc binding correctness.** Mitigation: copy the proven `.parameter set` escaping from `:2779`; verify with a test that includes a request_id containing an apostrophe.
  - **Low — `SELECT changes()` parsing** (must `tail -n1` the captured output, since the UPDATE may emit nothing). Mitigation: explicit U5 zero-row test.
