### TASK-001 — Wire `sync_intake_db_row` into `escalate_to_paused`

- **Description**: Implement the only behaviour-changing edit in this request.
  After the atomic `mv "${tmp}" "${state_file}"` inside `escalate_to_paused`,
  (a) read `current_phase` from the freshly-written authoritative state.json
  with a fallback to the caller-supplied `phase` argument when the read yields
  an empty string, and (b) invoke
  `sync_intake_db_row "${request_id}" "${current_phase}" "paused" "${ts}"`
  *after* the existing `events.jsonl` append and *before* the alert/log
  emission. Uses the same `$ts` already written to `state.json.updated_at` so
  ledger and state remain synchronised on the same timestamp.
  References: TDD §4.1 (pseudocode), §3.3 (data flow), ADR-2, ADR-3, ADR-4;
  PRD FR-A1, FR-A2, FR-A3.
- **Files**:
  - `plugins/autonomous-dev/bin/supervisor-loop.sh` (modify, around lines
    1678–1700; helper call insertion at ~line 1685)
- **Dependencies**: none.
- **Acceptance Criteria**:
  1. `escalate_to_paused` reads `current_phase` from `state.json` via
     `jq -r '.current_phase // empty'` after the `mv`.
  2. Falls back to the function's `$3` (`phase`) argument if the jq read
     yields an empty string or fails.
  3. Calls `sync_intake_db_row "${request_id}" "${current_phase}" "paused"
     "${ts}"` exactly once per pause.
  4. The call occurs after `mv` and after `events.jsonl` append, before the
     `emit_alert` / `log_error` lines (TDD §3.3 step ordering).
  5. No other behaviour change: existing jq filter, atomic write, alert,
     and log lines are byte-identical to pre-fix.
  6. `state.json` write path remains authoritative; ledger failure does not
     abort the pause (inherited from helper's non-fatal contract).
- **Lint/Test Commands**:
  - `bash -n plugins/autonomous-dev/bin/supervisor-loop.sh`
  - `shellcheck plugins/autonomous-dev/bin/supervisor-loop.sh` (warnings
    OK if pre-existing; no NEW shellcheck violations).
  - `bats plugins/autonomous-dev/tests/bats/escalate_to_paused.bats`
    (runs once TASK-004 lands; for TASK-001 itself, a syntax check + manual
    diff inspection is sufficient).
- **Estimated Effort**: 1.5 h (implementation 0.5 h + manual smoke
  verification 0.5 h + 20 % buffer 0.4 h).
- **Track**: A (critical path).
- **Risks**:
  - *Medium / fix*: Mis-ordering the helper call relative to the
    `events.jsonl` append could produce an event-but-no-ledger window on
    sqlite failure. Mitigation: TDD §3.3 explicitly fixes ordering;
    reviewer to verify by diff.
  - *Low / detection*: line-number drift may leave the AUDIT REFERENCE
    table (TASK-003) and the actual `mv` site out of sync. Mitigation:
    TASK-003 explicitly refreshes against HEAD.
