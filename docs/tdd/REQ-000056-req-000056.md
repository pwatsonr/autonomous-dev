## 2. Overview

This TDD operationalizes the PRD's principle — *every reachable failure mode
must (a) always be detected + recorded and (b) trigger an automatic remediation
or safe continue, with a human as the FINAL escalation*. The implementation
adds a single sourced bash module, `lib/self-heal.sh`, that exposes:

1. **A data-driven dispatch table** mapping `failure_mode → (detector_fn,
   event_type, remediation_fn, default_policy)`. The supervisor never
   hard-codes mode-specific logic; it iterates the table.
2. **Pure, idempotent detector functions** invoked at well-defined
   integration points in `supervisor-loop.sh` (`run_review_gate_phase`,
   `advance_phase`, `check_phase_timeout`, `dispatch_phase_session`
   return path).
3. **Pure, idempotent remediator functions** that own the only writes to
   `state.json.current_phase_metadata.self_heal.*`. Remediators NEVER
   touch `current_phase` or `status` except via the existing
   `escalate_to_paused` / `advance_phase` helpers.
4. **Universal fallback**: any exception inside a detector/remediator
   downgrades to `R_CAPTURE_AND_PAUSE`, guaranteeing the invariant that
   the supervisor never hard-fails without a paused checkpoint.
5. **Kill switch**: `AUTONOMOUS_DEV_SELF_HEAL=0` short-circuits every
   detector to a no-op so the supervisor reverts to current behavior
   bit-for-bit on the bypassed paths (NFR-COMPAT-02).

The design intentionally LAYERS on top of (does not replace) the existing
soft-timeout machinery (REQ-000051 / ADR-004), the reviewer-chain CLI
(`bin/review-gate.ts`), the ledger sync (REQ-000013/14), and
`escalate_to_paused`. No existing function is renamed; every integration
point is a new conditional that calls into `self-heal.sh` BEFORE the
existing behavior fires, with a documented "if remediation succeeded,
skip the existing path" contract.

The new TypeScript surface is minimal: `bin/review-gate.ts` gains a
per-reviewer retry-once-then-exclude policy delegated by a new exported
function in `intake/reviewers/runner.ts`. Everything else is bash + jq.

### 3.3 Data Flow (per failure mode)

For every failure mode F1..F9:

1. **Trigger point** (a specific call site in `supervisor-loop.sh`) hands
   off control to `selfheal_dispatch <F_ID> <ctx_json>`.
2. `selfheal_dispatch` reads the table row for `F_ID` and calls
   `detect_<F>` which returns `0` (detected) or `1` (not detected).
3. On detection, `detect_<F>` calls `selfheal_emit_event <type> <evidence>`
   which writes one line to `events.jsonl` AND validates the line against
   the schema in `docs/schemas/events/`.
4. The dispatcher then calls `remediate_<R>(ctx)`. The remediator returns:
   - `0` = remediated; supervisor SHOULD skip the legacy failure path.
   - `1` = remediation declined (e.g. budget already extended once).
   - `≥2` = remediation hard-failed; dispatcher falls back to `R_CAPTURE_AND_PAUSE`.
5. The dispatcher emits a follow-up `<remediation>_outcome` event.

## 4. Detailed Design

### 4.1 `lib/self-heal.sh` — Dispatch Table

The table is a data-driven array of TSV rows sourced at module load. Each
row is `mode_id\tdetector_fn\tevent_type\tremediator_fn\tdefault_policy`.

```bash
# self-heal.sh (pseudo-code)

# Public API
#   selfheal_dispatch <integration_point> <ctx_json>
#   selfheal_emit_summary <request_id> <project>
#   selfheal_is_enabled  (respects AUTONOMOUS_DEV_SELF_HEAL=0)

readonly SELF_HEAL_SCHEMA_DIR="${PLUGIN_DIR}/../../docs/schemas/events"

# Default thresholds (all overridable by AUTONOMOUS_DEV_SELF_HEAL_*)
: "${AUTONOMOUS_DEV_SELF_HEAL:=1}"
: "${AUTONOMOUS_DEV_SELF_HEAL_MIN_PHASE_DURATION_SECONDS:=5}"
: "${AUTONOMOUS_DEV_SELF_HEAL_REVIEW_LOOP_THRESHOLD:=3}"
: "${AUTONOMOUS_DEV_SELF_HEAL_REVIEWER_TIMEOUT_THRESHOLD:=2}"
: "${AUTONOMOUS_DEV_SELF_HEAL_BUDGET_EXTENSION_FACTOR:=1.5}"
: "${AUTONOMOUS_DEV_SELF_HEAL_REVIEWER_TIMEOUT_MULTIPLIER:=2}"
: "${AUTONOMOUS_DEV_SELF_HEAL_DIAG_BUNDLE_MAX_BYTES:=1048576}"  # 1 MiB
: "${AUTONOMOUS_DEV_SELF_HEAL_SUSPICIOUS_FAST_RATIO:=10}"

# Integration-point -> ordered list of mode IDs to attempt
_SELFHEAL_DISPATCH_REVIEW_OUTCOME=("F1" "F2" "F4")
_SELFHEAL_DISPATCH_SESSION_OUTCOME=("F5" "F6")
_SELFHEAL_DISPATCH_ADVANCE_PHASE=("F7" "F8")
_SELFHEAL_DISPATCH_PHASE_TIMEOUT=("F3")
_SELFHEAL_DISPATCH_LEDGER_CHECK=("F9")

# Table row resolver: prints "detector|event_type|remediator|policy_name"
_selfheal_table_lookup() {
    local mode="$1"
    case "$mode" in
        F1) printf '%s' "detect_review_gate_loop|review_gate_loop_detected|remediate_fallback_to_single_reviewer|R_FALL_BACK_TO_SINGLE_REVIEWER" ;;
        F2) printf '%s' "detect_reviewer_timeout_repeated|reviewer_timeout_repeated|remediate_escalate_reviewer_timeout|R_ESCALATE_REVIEWER_TIMEOUT" ;;
        F3) printf '%s' "detect_phase_timeout_with_progress|phase_timeout_with_progress|remediate_extend_phase_budget|R_EXTEND_PHASE_BUDGET" ;;
        F4) printf '%s' "detect_reviewer_error|reviewer_error_detected|remediate_retry_then_exclude|R_RETRY_ONCE_THEN_EXCLUDE_IF_NON_BLOCKING" ;;
        F5) printf '%s' "detect_suspicious_empty|suspicious_empty_result|remediate_requeue_author_phase_once|R_REQUEUE_AUTHOR_PHASE_ONCE" ;;
        F6) printf '%s' "detect_suspicious_fast|suspicious_fast_result|remediate_requeue_author_phase_once|R_REQUEUE_AUTHOR_PHASE_ONCE" ;;
        F7) printf '%s' "detect_verification_false_negative|verification_false_negative_detected|remediate_self_verify|R_SELF_VERIFY" ;;
        F8) printf '%s' "detect_novel_failure|novel_failure_detected|remediate_capture_and_pause|R_CAPTURE_AND_PAUSE" ;;
        F9) printf '%s' "detect_state_ledger_drift|state_ledger_drift_detected|remediate_reconcile_ledger|R_RECONCILE_LEDGER" ;;
        *)  return 1 ;;
    esac
}
```

#### Dispatcher contract

```bash
# selfheal_dispatch(integration_point: string, ctx_json: string) -> int
#   integration_point ∈ {review_outcome, session_outcome, advance_phase,
#                        phase_timeout, ledger_check}
#   ctx_json: a serialized JSON blob carrying {request_id, project, phase,
#             result_file?, gate_decision?, pre_tree?, post_tree?, ...}
#
# Returns:
#   0  — at least one remediation succeeded; caller SHOULD skip legacy fail
#         path.
#   1  — no detection fired; caller continues with legacy path unchanged.
#   2  — fatal exception during dispatch; R_CAPTURE_AND_PAUSE was invoked.
#         Caller MUST treat as "paused" and not run additional failure logic.
selfheal_dispatch() {
    local integration_point="$1" ctx_json="$2"
    selfheal_is_enabled || return 1

    local mode_ids_var="_SELFHEAL_DISPATCH_$(echo "$integration_point" | tr '[:lower:]' '[:upper:]')[@]"
    local mode_ids=("${!mode_ids_var}")

    local any_remediated=1
    for mode in "${mode_ids[@]}"; do
        IFS='|' read -r detector event_type remediator policy <<< "$(_selfheal_table_lookup "$mode")"

        # All detectors MUST return 0 if detected, 1 if not. Errors -> R_CAPTURE_AND_PAUSE.
        if ! _selfheal_safe_call "$detector" "$ctx_json"; then
            local detect_rc=$?
            if [[ $detect_rc -ge 2 ]]; then
                remediate_capture_and_pause "$ctx_json" "$mode" "detector_threw"
                return 2
            fi
            continue
        fi

        selfheal_emit_event "$event_type" "$ctx_json" "$policy"

        if _selfheal_safe_call "$remediator" "$ctx_json"; then
            selfheal_emit_event "${policy}_succeeded" "$ctx_json"
            any_remediated=0
        else
            local rc=$?
            if [[ $rc -ge 2 ]]; then
                remediate_capture_and_pause "$ctx_json" "$mode" "remediator_threw"
                return 2
            fi
            selfheal_emit_event "${policy}_declined" "$ctx_json"
        fi
    done

    return "$any_remediated"
}

# _selfheal_safe_call wraps a function call with `(set +e; fn args)` so
# `set -euo pipefail` in the supervisor never crashes mid-detect.
```

### 4.2 Detector Functions (specifications)

Each detector reads from `ctx_json` (via `jq`) and must NOT mutate state.
All return `0` for detected, `1` for not detected, `≥2` for unrecoverable
internal error (caught by `_selfheal_safe_call`).

#### detect_review_gate_loop (F1, #615)

```
Inputs (ctx_json): request_id, project, phase, gate_decision (full JSON
                   from review-gate.ts), state_file
Algorithm:
  1. reviewer  ← jq '.findings[0].reviewer // empty' on gate_decision
  2. reason    ← jq '.reason // .findings[0].reason // empty'
  3. outcome   ← jq '.outcome'
  4. Return 1 if outcome != "REQUEST_CHANGES" or reviewer empty.
  5. Load prior loop record from
     state.current_phase_metadata.self_heal.review_loop[<reviewer>]:
       { count: int, last_reason_fingerprint: string,
         last_seen_at: ISO8601 }
  6. fingerprint ← sha256(reason | head -c 256)
  7. If fingerprint == last_reason_fingerprint, increment count;
     else reset count=1, set fingerprint.
  8. Persist via selfheal_state_set "review_loop.<reviewer>" <updated>
     (atomic — see §4.4).
  9. Return 0 iff count >= REVIEW_LOOP_THRESHOLD (default 3).

Evidence emitted: { reviewer, fingerprint, count, last_seen_at }
```

#### detect_reviewer_timeout_repeated (F2, #618/#620)

```
Inputs: request_id, project, phase, gate_decision (with synthesized
        status:"error" or "timeout"), reviewer_name, state_file
Algorithm:
  1. Return 1 if gate_decision.status not in {error,timeout} OR
     gate_decision.error_kind != "reviewer_timeout".
  2. Read state.current_phase_metadata.self_heal.reviewer_timeouts[<rev>]
     (int, default 0); increment.
  3. Persist.
  4. Return 0 iff updated >= REVIEWER_TIMEOUT_THRESHOLD (default 2).

Evidence: { reviewer, timeout_count, current_timeout_seconds }
```

#### detect_phase_timeout_with_progress (F3, #616)

```
Inputs: request_id, project, phase, pre_tree, post_tree
Algorithm:
  1. If working_tree_advanced "$pre_tree" "$post_tree" == 0
     (i.e. progress detected) AND check_phase_timeout returned 1
     (hard timeout fired), return 0.
  2. Else return 1.

Evidence: { pre_tree, post_tree, elapsed_seconds, timeout_seconds }

Note: F3 ONLY runs when invoked from the hard-timeout path; the existing
soft-timeout path is untouched.
```

#### detect_reviewer_error (F4, #618)

```
Inputs: result_file (phase-result-<phase>.json), state_file, reviewer_name
Algorithm:
  1. status_reason ← jq '.status_reason // empty' on result_file
  2. status        ← jq '.status'
  3. Return 1 unless status == "error" AND
     status_reason ∈ {REVIEW_GATE_CLI_NONZERO, REVIEW_GATE_BAD_JSON,
                       REVIEW_GATE_RESULT_WRITE_FAILED}.
  4. Return 0 otherwise.

Evidence: { reviewer_name, status_reason, exit_code, stdout_head: first 1KB }
```

#### detect_suspicious_empty (F5)

```
Inputs: result_file, session_log_path, phase, state_file
Algorithm:
  1. If phase ∉ {prd, tdd, plan, spec, code}, return 1.
  2. If jq '.status' != "pass", return 1.
  3. artifact_count ← jq '(.artifacts // []) | length'
  4. session_duration ← mtime(result_file) − mtime(session_log_path)
  5. Return 0 iff artifact_count == 0 AND
     session_duration < MIN_PHASE_DURATION_SECONDS.

Evidence: { phase, artifact_count, session_duration_seconds, threshold }
```

#### detect_suspicious_fast (F6)

```
Inputs: result_file, session_log_path, phase, request_type, state_file
Algorithm:
  1. session_duration ← mtime(result_file) − mtime(session_log_path)
  2. baseline ← read median duration for (phase, request_type) from
                cached telemetry file at
                ${DAEMON_HOME}/self-heal/phase-duration-medians.json
                (best-effort; treat missing as 0).
  3. If baseline == 0, return 1 (no baseline yet — cannot judge).
  4. Return 0 iff session_duration < (baseline / SUSPICIOUS_FAST_RATIO)
                AND session_duration < MIN_PHASE_DURATION_SECONDS.

Evidence: { phase, session_duration_seconds, baseline_seconds, ratio }
```

#### detect_verification_false_negative (F7, #617)

```
Inputs: result_file, project, phase
Algorithm:
  1. If jq '.status' on result_file != "fail", return 1.
  2. reason ← jq '.status_reason // ""'
  3. Try to locate referenced artifact:
       (a) jq '.test_results_path // .test_artifact // empty'
       (b) if (a) empty, scan reason for "test-results.json" path tokens
       (c) if still empty, default to
           "${project}/.autonomous-dev/requests/<id>/test-results.json"
  4. If artifact missing OR mtime older than phase_started_at, return 1
     (cannot conclude false-negative; emit no event).
  5. failed_count ← jq '.failures // .stats.failures // ([.tests[]?|select(.status=="failed")] | length)'
  6. Return 0 iff failed_count == 0.

Evidence: { artifact_path, artifact_mtime, failures, original_status,
            original_status_reason }
```

#### detect_novel_failure (F8)

```
Inputs: result_file, state_file
Algorithm:
  1. status ← jq '.status // ""' on result_file (or "")
  2. If status ∈ {pass, fail, error, timeout}, return 1.
  3. OR: if state.json fails jq parse, return 0 (novel).
  4. OR: if jq '.current_phase' references an artifact path that doesn't
         exist, return 0.

Evidence: { observed_status, state_parseable, missing_artifact_paths }
```

#### detect_state_ledger_drift (F9)

```
Inputs: request_id, state_file
Algorithm:
  1. state_status ← jq '.status' on state_file
  2. db_status   ← sqlite3 "$INTAKE_DB" \
                     "SELECT status FROM requests WHERE id='$request_id'"
  3. Return 0 iff state_status != db_status (and both non-empty).

Evidence: { state_status, db_status }
```

### 4.3 Remediator Functions (specifications)

All remediators MUST be idempotent per (request, phase) per FR-REM-01.
Idempotency is enforced by reading state flags BEFORE acting and bailing
with rc=1 if the flag already set.

#### remediate_fall_back_to_single_reviewer (R for F1)

```
Behavior:
  1. If state.current_phase_metadata.self_heal.review_chain_disabled == true,
     return 1 (already done).
  2. Set self_heal.review_chain_disabled = true,
         self_heal.review_chain_disabled_at = NOW,
         self_heal.review_chain_disabled_for_phase = <phase>.
  3. Emit alert "review_chain_disabled_for_request".
  4. Return 0.

Effect on next dispatch:
  should_use_review_chain() (existing helper) must consult this flag —
  see §4.6 modification list.
```

#### remediate_escalate_reviewer_timeout (R for F2)

```
Behavior:
  1. If self_heal.reviewer_timeout_extended[<reviewer>] == true, fall
     through to R_FALL_BACK_TO_SINGLE_REVIEWER.
  2. Else: set extended flag, persist
     reviewer_timeout_seconds * MULTIPLIER (default 2×) into
     self_heal.reviewer_timeout_overrides[<reviewer>].
  3. Emit "reviewer_timeout_extended" event.
  4. Return 0.

Wiring: review-gate-cli.ts reads
        self_heal.reviewer_timeout_overrides on next invocation.
```

#### remediate_extend_phase_budget (R for F3, #616)

```
Behavior:
  1. If self_heal.budget_extended_at != null, return 1.
  2. resolved_timeout ← resolve_phase_timeout state_file phase
  3. new_timeout ← floor(resolved_timeout * BUDGET_EXTENSION_FACTOR)
  4. Persist:
       self_heal.budget_extended_at = NOW
       self_heal.budget_extended_from = resolved_timeout
       self_heal.budget_extended_to   = new_timeout
       phase_started_at = NOW (so the existing check_phase_timeout
                               re-arms with the new clock)
  5. Emit "phase_budget_extended".
  6. Return 0.

Note: resolve_phase_timeout must be amended (§4.6) to honor
self_heal.budget_extended_to as an override.
```

#### remediate_retry_then_exclude (R for F4, #618)

```
Behavior (split across bash + TS):

Bash side:
  1. If self_heal.reviewer_retry[<reviewer>] not set: set it to 1, emit
     "reviewer_retry_scheduled", return 0 (caller re-dispatches review).
  2. If reviewer_retry[<reviewer>] == 1 (already retried once):
     blocking ← jq '.reviewers[]|select(.name=="<rev>")|.blocking' on
                 the reviewer-chain config (see §4.7 / OQ-1 resolution).
     If blocking == true:
       Emit "reviewer_failed_blocking" → return 1 (caller fails the gate).
     Else:
       Add <reviewer> to self_heal.excluded_reviewers[].
       Emit "reviewer_excluded".
       Return 0 (caller re-dispatches review with reviewer excluded).

TS side (runner.ts):
  - runReviewGate honors a new ExclusionList argument (string[] of
    reviewer names) read by review-gate-cli from state.json on entry.
```

#### remediate_self_verify (R for F7, #617)

```
Behavior:
  1. failures ← (computed by detector; carried in ctx_json)
  2. If failures > 0, return 1 (real failure; do not override).
  3. Rewrite result_file atomically: jq '.status = "pass" |
       .self_verified = true | .original_status = "fail" |
       .self_verified_at = NOW' >tmp && mv tmp result_file
  4. CRITICAL: retry_count MUST NOT be incremented. Caller path
     (advance_phase) must check .self_verified and bypass the "fail"
     escalation_count++ branch. See §4.6 modification list.
  5. Emit "verification_false_negative_corrected".
  6. Return 0.

Stale-artifact guard: detector already filters by mtime ≥ phase_started_at.
Mitigates Risk #2 (R_SELF_VERIFY flips a real failure).
```

#### remediate_requeue_author_phase_once (R for F5/F6)

```
Behavior:
  1. If self_heal.requeued[<phase>] == true, fall through to
     R_CAPTURE_AND_PAUSE (suspicious twice = genuinely broken).
  2. Persist self_heal.requeued[<phase>] = true,
            self_heal.requeue_reason = "suspicious_<empty|fast>".
  3. Move result_file aside to result_file.suspicious-<ts>.bak.
  4. Reset state.json:
        .status = "running"
        .current_phase = <phase>            (unchanged)
        .current_phase_metadata.dispatched_phase = null
        .current_phase_metadata.suspicious_previous_result = true
     (this hint is read by resolve_phase_prompt and prepended to the
      next session — see §4.6).
  5. DO NOT increment retry_count.
  6. Emit "phase_requeued_suspicious".
  7. Return 0.
```

### 4.4 `lib/self-heal-state.sh` — Atomic State Writers

A thin wrapper that all remediators MUST use to touch
`state.json.current_phase_metadata.self_heal.*`.

```bash
# selfheal_state_set(state_file, dot_path, json_value) -> int
#   Atomically merges {dot_path : json_value} into
#   .current_phase_metadata.self_heal.<dot_path>.
#   Uses tmp+rename (TC-06). Returns 0 on success, 2 on jq failure.
selfheal_state_set() {
    local state_file="$1" path="$2" value="$3"
    local tmp="${state_file}.tmp.$$"
    if jq --arg p "$path" --argjson v "$value" \
        '.current_phase_metadata.self_heal //= {} |
         .current_phase_metadata.self_heal |= (
            . as $cur | $cur | setpath(($p|split(".")); $v)
         )' "$state_file" > "$tmp" 2>/dev/null; then
        mv "$tmp" "$state_file"
        return 0
    fi
    rm -f "$tmp" 2>/dev/null || true
    return 2
}
```

## 7. Error Handling

### 7.1 Error Taxonomy

| Error class | Source | Classification | Handling |
|---|---|---|---|
| Detector internal exception (e.g. jq error) | bash | Recoverable | `_selfheal_safe_call` catches via subshell; treat as "not detected"; `log_warn` |
| Remediator hard failure (state write fails) | bash | Critical | Falls through to `R_CAPTURE_AND_PAUSE` |
| Schema validation failure | self-heal-events.sh | Log-only | Event STILL appended (visibility > purity), `log_warn` only |
| Diagnostic bundle write failure | `remediate_capture_and_pause` | Critical | `log_error`; call legacy `escalate_to_paused` directly with `paused_reason: "novel_failure_bundle_failed"` |
| `gh` CLI absent or fails | `remediate_capture_and_pause` | Soft | Bundle still written; `issue_filed_url: null` |
| Reviewer config missing `blocking` field | F4 remediator | Recoverable | Treat as `blocking: true` (conservative default); emit `reviewer_blocking_default_assumed` event |
| `AUTONOMOUS_DEV_SELF_HEAL=0` set | dispatcher | Bypass | All hooks return 1 (no-op); legacy paths run untouched |

### 7.2 User-Facing Error Messages

Self-healing surfaces results to the operator via the same channels as
existing failures (`events.jsonl`, alerts dir, `paused_reason`). No new
TUI; portal/dashboards consume the new event types.

Sample `paused_reason` strings (for the operator viewing the portal):

- `"novel_failure: <mode_id> (see diagnostic-bundle-<ts>.json)"`
- `"reviewer_chain_disabled_and_single_reviewer_still_blocked"`
- `"suspicious_previous_result: requeue exceeded"`

## 9. Security Considerations

| Concern | Mitigation |
|---|---|
| Diagnostic bundles leaking secrets via env vars | Explicit allow-list `SELFHEAL_ENV_ALLOWLIST` (NFR-SECURITY-01); regex-asserted in `R_capture_and_pause.bats` |
| Diagnostic bundles leaking secrets via session log | Reuse the existing log-redaction filter (`scrub_secrets()` if present, else extend); truncate to last 200 lines (FR-REM-06) |
| Issue-filing exposing internal paths | The issue body uses path basenames only (no absolute paths) |
| State writes racing across `selfheal_state_set` and supervisor mutations | All writes go through tmp+rename (TC-06); reads tolerate stale views since each mutation is idempotent (FR-REM-01) |
| `events.jsonl` corruption from concurrent appends | `flock` on `events.jsonl.lock`; appending is the only operation (no truncate) |
| Token / `gh` CLI privilege escalation when filing novel-failure issues | `gh` runs only if `AUTONOMOUS_DEV_SELF_HEAL_FILE_ISSUES=1` AND `gh auth status` exits 0; otherwise bundle-only (N4 honored) |
| F7 flipping a real failure due to stale test artifact | Mtime ≥ `phase_started_at` guard in detector; covered by `F7_stale_artifact.bats` (Risk #2 mitigation) |

## 10. Performance Considerations

| Budget | Source | Strategy |
|---|---|---|
| Detection overhead ≤ 50 ms per phase transition (NFR-PERF-01) | PRD | Detectors are jq one-shots (~5–10 ms each); 9 detectors × 5 ms ≈ 45 ms median. Lazy short-circuit on early `return 1`. |
| Diagnostic bundle ≤ 500 ms for 200-line log (NFR-PERF-02) | PRD | `tail -n 200` + `jq -nc` ≈ 30–80 ms on typical hosts. CI bench gated by `R_capture_and_pause.bats`. |

| State writes under contention | Design | All writes are `tmp+rename`; readers see atomic snapshot. |
| Schema validation overhead | Design | `ajv` validation gated by `AUTONOMOUS_DEV_SELF_HEAL_VALIDATE_SCHEMA` (default 1 in test envs, 0 in production deployments by daemon config). |

## 11. Architecture Decision Records (ADRs)

### ADR-1: Data-driven dispatch table vs. inline conditionals

- **Context.** PRD requires adding a failure mode in ≤3 files (NFR-MAINTAINABILITY-01) and TC-07 mandates data-driven dispatch.
- **Decision.** Single TSV-like lookup function `_selfheal_table_lookup` + per-integration-point ordered ID arrays.
- **Alternatives considered.** (a) Inline switch in supervisor-loop.sh (rejected — violates NFR-MAINTAINABILITY-01). (b) External JSON config (rejected — adds runtime dep on jq schema parsing and complicates bats fixtures).
- **Consequences.** Adding a new mode is exactly: 1 case branch + 1 detector fn + 1 remediator fn + 1 bats file + 1 schema.

### ADR-2: Self-heal state lives under `current_phase_metadata.self_heal` (not top-level)

- **Context.** PRD N2 forbids breaking schema changes; existing daemons must ignore new fields.
- **Decision.** Nested sub-object under `current_phase_metadata`.
- **Alternatives.** Top-level `self_heal` field (rejected — broader surface; harder to reason about lifetime).
- **Consequences.** Cleared on phase advance via the existing `dispatched_phase = null` reset — EXCEPT for cross-phase fields. We add an explicit preserve-list (`excluded_reviewers`, `review_chain_disabled`) to the existing `jq` resets in `advance_phase`.

### ADR-3: Bash + jq, no new runtime deps

- **Context.** TC-05 forbids new third-party deps.
- **Decision.** Detectors/remediators are bash; only TS surface is the reviewer runner (already TS).
- **Alternatives.** Pull self-heal into TS (rejected — supervisor-loop.sh is the integration surface; round-tripping via TS adds startup cost).
- **Consequences.** Slightly more verbose detector code; bats coverage is the trade-off.

### ADR-4: Schema validation gated by env, but events ALWAYS appended

- **Context.** NFR-OBSERVABILITY-01 requires schemas; FR-DETECT-05 requires unconditional event writes.
- **Decision.** Validate when `AUTONOMOUS_DEV_SELF_HEAL_VALIDATE_SCHEMA=1` (CI default); log + still write on violation.
- **Alternatives.** Hard-fail on schema mismatch (rejected — violates FR-DETECT-05).
- **Consequences.** Schema drift is a CI failure (not a runtime failure), preserving the visibility-over-purity principle.

### ADR-5: F7 (self-verify) is single-pass per result_file

- **Context.** S-OP-2 and FR-REM-04 require not consuming retry budget; Risk #2 warns against stale-artifact false flips.
- **Decision.** Detector requires `artifact_mtime ≥ phase_started_at`; remediator rewrites `result_file` once with `self_verified: true`; `advance_phase` checks that flag and bypasses the escalation_count++ path.
- **Alternatives.** Re-run the test command before override (deferred to a follow-up; tracked in OQ-2 resolution path).
- **Consequences.** A genuinely intermittent test bug stays masked for one cycle; F7 fires every cycle so the event stream surfaces the pattern.

### ADR-6: Kill switch default = ON (= self-healing ENABLED)

- **Context.** OQ-5; PRD Risk "changes phase-failure semantics".
- **Decision.** Default `AUTONOMOUS_DEV_SELF_HEAL=1`. The "v1 opt-in, v2 default-on" path is replaced by "v1 default-on with kill switch" since #620 is explicit that human escalation is FINAL.
- **Alternatives.** Default-off in v1 (rejected — defeats the PRD principle; new operators get worst-of-both-worlds).
- **Consequences.** Operators relying on the current "fast fail" must set `AUTONOMOUS_DEV_SELF_HEAL=0` explicitly. Documented in the migration note (§14).

### ADR-7: Reviewer "blocking" attribute lives in the chain config

- **Context.** OQ-1.
- **Decision.** Each entry in `intake/reviewers/chains/<request_type>.json` (the existing reviewer-chain config) gains an explicit `blocking: boolean` field with default `true`.
- **Alternatives.** Infer from reviewer name (rejected — magic strings; brittle).
- **Consequences.** Chain config schema bumps by one optional field; existing entries default to `blocking: true` (conservative; preserves today's gate-blocking behavior).

### ADR-8: Issue filing is opportunistic and gated by an env flag

- **Context.** PRD N4.
- **Decision.** `gh issue create` runs only if `gh auth status` succeeds AND `AUTONOMOUS_DEV_SELF_HEAL_FILE_ISSUES=1` AND a `--repo` target is resolvable; otherwise bundle is written but no remote issue is created.
- **Alternatives.** Always attempt issue creation (rejected — N4); never file (rejected — loses operator visibility).
- **Consequences.** Default behavior is bundle-only (safe); operators with `gh` configured can opt into filing.

## 13. Open Questions

1. **OQ-1 → ADR-7 (resolved).** `blocking` is an explicit field in the
   chain config; default `true`.
2. **OQ-2 → ADR-5 (partial).** Test-result discovery falls back to
   `<request>/.autonomous-dev/requests/<id>/test-results.json` then
   path-token scan of `status_reason`. Re-running the test command on
   missing artifacts is deferred to a follow-up — F7 today emits
   `self_verify_skipped_missing_artifact` and propagates `fail`.
3. **OQ-3 (open).** Suspicious-fast baseline uses per-phase median
   across all request_types in v1 to keep the cache small; per-(phase,
   request_type) is a future enhancement.
4. **OQ-4 → ADR-8 (resolved).** Default target for opportunistic issue
   filing is the autonomous-dev repo, overridable via
   `AUTONOMOUS_DEV_SELF_HEAL_ISSUE_REPO`.
5. **OQ-5 → ADR-6 (resolved).** Default ON with kill switch.
6. **OQ-6 (open).** `emit_alert` rate-limiting behavior to be
   confirmed; until then, `selfheal_emit_event` writes events
   independently of `emit_alert`, so self-heal events are NOT
   rate-limited even if alerts are.
7. **OQ-7 → ADR-fallback (resolved).** Diagnostic bundle cap = 1 MiB
   default; overridable via
   `AUTONOMOUS_DEV_SELF_HEAL_DIAG_BUNDLE_MAX_BYTES`.

### Remaining open spike items for the spec phase

- **OQ-S1.** Exact fingerprint algorithm for F1 — sha256 of the first
  256 bytes of `.reason` is the proposed default; reviewers may want
  token-level normalization.
- **OQ-S2.** How `selfheal_emit_summary` deduplicates when a request
  has multiple terminal transitions (e.g., `paused → done` after resume) —
  current proposal is "emit once per terminal entry; portal can sum".

## 14. Backward Compatibility & Migration

- **No state.json migration required.** All additions are nested under
  `current_phase_metadata.self_heal.*` (additive).
- **No intake.db migration required.** F9 uses existing helper.
- **events.jsonl** gains 15 new event types; consumers must tolerate
  unknown `type` values (existing portal/dashboard code already
  iterates with `case ... default: noop`).
- **`should_use_review_chain` contract change.** Now consults
  `self_heal.review_chain_disabled` — but only when self-healing is on.
  With `AUTONOMOUS_DEV_SELF_HEAL=0` the helper is bit-for-bit identical.
- **Reviewer chain config schema** gains an optional `blocking` field
  (default `true`). Older configs continue to work — they are treated
  as blocking, the conservative default.
- **Operator migration note** (to ship in T12 docs): users relying on
  current "fast fail" semantics can pin
  `AUTONOMOUS_DEV_SELF_HEAL=0` in `autonomous-dev.json`'s daemon env.
