  fallback + warning.

  strictly `=== '1'`; ANY other value (including `'true'`, `'yes'`,
  `'TRUE'`) → `false`, and if the env var is present AND the value is
  not `'0'` and not `'1'`, a `ConfigWarning` is emitted.

### 1.3 `ledger.ts` (TASK-003)

```ts
export type LedgerStatus = 'idle' | 'backoff' | 'in_flight' | 'capped';
export type LedgerOutcome = 'success' | 'failed' | 'cancelled' | 'unknown';

export interface LedgerEntry {
  repoId: string;
  issueNumber: number;
  issueFingerprint: string | null;
  requestIds: string[];
  attempts: number;
  lastAttemptAt: string;      // ISO-8601
  lastOutcome: LedgerOutcome;
  backoffUntil: string | null;
  status: LedgerStatus;
}

export interface WindowCost {
  totalUsd: number;
  requestCount: number;
}

export interface LedgerFile {
  version: 1;
  entries: Record<string, LedgerEntry>;  // key: `${repoId}#${issueNumber}`
  windowCosts: Record<string, WindowCost>; // key: ISO hour, e.g. '2026-07-01T14'
  loadWarnings?: ConfigWarning[];        // populated when file was corrupt
}

export interface LedgerIO {
  homedir(): string;
  readFile(p: string): string | undefined;
  writeFile(p: string, data: string): void;
  mkdirp(p: string, mode: number): void;
  chmod(p: string, mode: number): void;
  openExclusive(p: string): number;      // returns fd; throws EEXIST on collision
  closeAndUnlink(fd: number, p: string): void;
  statMtimeMs(p: string): number | null; // null if not present
  now(): number;
  randSuffix(): string;                  // e.g. `${pid}.${random}`
}

export function ledgerPath(io: LedgerIO): string;   // ~/.autonomous-dev/state/self-improve/ledger.json
export function lockPath(io: LedgerIO): string;     // ~/.autonomous-dev/state/self-improve/ledger.lock
export function loadLedger(io: LedgerIO): LedgerFile;
export function saveLedger(f: LedgerFile, io: LedgerIO): void;

export interface LedgerReader {
  getEntry(key: string): LedgerEntry | undefined;
  getInFlightAutoFixRequest(key: string): string | undefined;
  countActiveGlobal(): number;
  countActivePerRepo(repoId: string): number;
  costLast24h(): number;
  costLast7d(): number;
}
export function makeReader(f: LedgerFile, cfg: SelfImproveConfig, now: number): LedgerReader;

export interface LedgerMutator {
  recordSubmission(key: string, entry: LedgerEntry): void;
  recordOutcome(key: string, outcome: 'success'|'failed'|'cancelled', costUsd: number): void;
  reset(key: string): void;
  reconcile(openIssueKeys: Set<string>): void;
  snapshot(): LedgerFile;
}
export function makeMutator(f: LedgerFile, cfg: SelfImproveConfig, now: number): LedgerMutator;
```

**Path / permission invariants:**
- `ledgerPath(io) === path.join(io.homedir(), '.autonomous-dev', 'state', 'self-improve', 'ledger.json')`.
- Parent directory created with mode `0o700` if absent.
- File written with mode `0o600`.

**Atomic write sequence in `saveLedger`:**
1. `io.mkdirp(parent, 0o700)`.
2. Acquire lock: loop calling `io.openExclusive(lockPath)` with 25 ms
   sleep, up to 500 ms total. On failure, if
   `io.statMtimeMs(lockPath)` is > 60 000 ms older than `io.now()`,
   force-delete stale lock and retry once more. Ultimate failure →
   throw `LedgerLockBusyError`.
3. Write to `${ledgerPath}.tmp.${io.randSuffix()}` as JSON (2-space).
4. `chmod(tmpPath, 0o600)`.
5. `rename` (atomic) to `ledgerPath` via `io.writeFile`+`rename` seam
   (implementation may split into two IO methods; keep atomic effect).
6. `io.closeAndUnlink(fd, lockPath)`.

**Corrupt file handling in `loadLedger`:**
- If file absent → return `{ version: 1, entries: {}, windowCosts: {} }`.
- If JSON parse fails: copy raw contents to
  `${ledgerPath}.corrupt-${new Date(io.now()).toISOString()}-${io.randSuffix()}`
  with mode `0o600`; return an empty ledger with
  `loadWarnings: [{ envVar: 'LEDGER_FILE', raw: '<truncated>', fallback: 'empty-ledger' }]`.

**Reader arithmetic:**
- `countActiveGlobal()` counts entries where `status === 'in_flight'`.
- `countActivePerRepo(repoId)` counts entries where `status === 'in_flight' && entry.repoId === repoId`.
- `costLast24h()` sums `windowCosts[hourKey].totalUsd` for all
  hourKeys satisfying `parseHourKeyToMs(k) >= now - 24*3600*1000`.
- `costLast7d()` analogous, 7 days.

**Hour key format:** `YYYY-MM-DDTHH` (UTC), no colons, no minutes.
E.g. `2026-07-01T14`. `parseHourKeyToMs` treats the key as the START
of the hour in UTC.

**Key validation:** `${repoId}#${issueNumber}` — `repoId` MUST match
`/^[A-Za-z0-9_.-]+\/[A-Za-z0-9_.-]+$/`; `issueNumber` MUST be a
positive integer string. Otherwise `recordSubmission` throws
`LedgerKeyInvalidError`.

**Reconcile:** for every ledger key NOT in `openIssueKeys`, set
`status: 'idle'`; if `lastOutcome === undefined` → `'unknown'`. Do
NOT modify `attempts`.

### 1.5 `evidence.ts` (TASK-005)

```ts
export interface EvidenceCheck {
  ok: boolean;
  reason?: string;
  detail?: unknown;
}

export interface EvidenceDeps {
  readState: (repoPath: string, requestId: string) => Promise<{ status?: unknown } | null>;
  fetchIssueEvents: (repoId: string, issueNumber: number) => Promise<IssueEventsSnapshot>;
  readReviewerBlock?: (repoId: string, issueNumber: number, fp: string) =>
    Promise<{ verdict: 'REQUEST_CHANGES' | 'ERROR' | 'APPROVE' | 'COMMENT' } | null>;
  timeoutMs: number;                // default 500
  botLogin: string;
}

export interface Ownership {
  repos: Array<{ repoId: string; path: string; enrolled: boolean }>;
}

export async function checkEvidence(
  klass: ActionableClassId,
  issue: IssueSnapshot,
  ownership: Ownership,
  deps: EvidenceDeps,
): Promise<EvidenceCheck>;
```

**A1 behavior (marker present already required by classifier):**
1. Locate `repoPath` from `ownership.repos.find(r => r.repoId === issue.repoId)?.path`;
   if none → `{ ok: false, reason: 'NA7_NO_REPO_PATH' }`.
2. Extract candidate request ids from `issue.body` using regex
   `/REQ-\d{6}/g`; take FIRST match. If no matches AND
   `issue.fingerprint` present → `{ ok: true, detail: { mode: 'marker-only' } }`.
3. Call `deps.readState(repoPath, reqId)` under timeout race. If it
   returns non-null with `status === 'failed'` →
   `{ ok: true, detail: { mode: 'state-verified', requestId: reqId } }`.
4. Otherwise → `{ ok: false, reason: 'NA7_STATE_MISMATCH' }`.

**A2 behavior:**
1. Require `issue.reviewerBlockFp`; missing → `{ ok: false, reason: 'NA7_NO_REVIEWER_BLOCK' }`.
2. If `deps.readReviewerBlock` absent → `{ ok: true, detail: { mode: 'marker-only' } }`.
3. Otherwise call it; `verdict === 'REQUEST_CHANGES'` → `{ ok: true }`.
4. `verdict === 'ERROR'` → `{ ok: false, reason: 'NA7_REVIEWER_ERROR' }`.
5. Any other verdict / null → `{ ok: false, reason: 'NA7_REVIEWER_VERDICT_UNKNOWN' }`.

**A3 behavior:**
1. Call `deps.fetchIssueEvents(issue.repoId, issue.number)`.
2. Get `actor = events.labeledBy[LABEL_AUTO_FIX]`.
3. Missing → `{ ok: false, reason: 'NA7_NO_AUTO_FIX_LABEL_EVENT' }`.
4. `actor === deps.botLogin` → `{ ok: false, reason: 'NA7_BOT_LABELER' }`.
5. Otherwise → `{ ok: true, detail: { humanLabeler: actor } }`.

**Timeout:** every awaited call is raced against
`new Promise((_, rej) => setTimeout(() => rej(new Error('EVIDENCE_TIMEOUT')), deps.timeoutMs))`.
Timeout error → `{ ok: false, reason: 'EVIDENCE_TIMEOUT' }`.

### 1.7 `gh_issues.ts` (TASK-007)

```ts
import type { ExecFn } from '../checks_client';

export interface ListOpenResult {
  issues: IssueSnapshot[];
  truncated: boolean;   // true iff more than 200 issues available
}

export interface GhIssueClient {
  listOpen(repoId: string, labels: readonly string[], limit: number): Promise<ListOpenResult>;
  getEvents(repoId: string, issueNumber: number): Promise<IssueEventsSnapshot>;
  comment(repoId: string, issueNumber: number, body: string): Promise<void>;
```

**`listOpen` implementation contract:**
- Invoke `exec(['gh', 'api', '/repos/${repoId}/issues?state=open&per_page=100&labels=${labels.join(',')}', '-i'])`.
  Use `-i` flag to include response headers so we can read `Link`.
- Parse JSON body; each issue mapped to `IssueSnapshot`:
  - `fingerprint`: first regex capture from body of
    `/<!--\s*autodev-failure:\s*([A-Fa-f0-9]{8,})\s*-->/`; else `null`.
  - `reviewerBlockFp`: first regex capture from body of
    `/<!--\s*autodev-reviewer:\s*([A-Za-z0-9_.-]+)\s*-->/`; else `null`.
  - `authorLogin`: `issue.user.login`.
  - `labels`: `issue.labels.map(l => l.name)`.
- If response has ≥100 items AND `Link` header contains
  `rel="next"`, fetch page 2 the same way. Beyond page 2, stop and
  set `truncated: true`.
- Argv MUST be passed as array; NEVER interpolate into a shell string.

**`getEvents`:**
- `exec(['gh', 'api', '/repos/${repoId}/issues/${issueNumber}/events'])`.
- Parse JSON; reduce to `IssueEventsSnapshot` where
  `labeledBy[label] = actor.login` — for events with `event === 'labeled'`,
  latest actor wins (iterate array in ascending `created_at`, later
  overwrites earlier).

**`comment`:**
- `exec(['gh', 'issue', 'comment', String(issueNumber), '--repo', repoId, '--body', body])`.

Error handling: any non-zero exit from `exec` → rejected promise
whose `.message` includes the stderr text. Caller (scan orchestrator)
maps to `GH_LIST_FAILED` / `GH_EVENTS_FAILED` / `GH_COMMENT_FAILED`.

```ts

}

export function buildSubmitPayload(
  issue: IssueSnapshot,

```

**Composition order in `description` (exact, joined with `\n\n`):**
1. `Auto-generated from ${issue.repoId}#${issue.number}: ${issue.title}`
2. `Source: ${issue.htmlUrl}`
3. Fenced verbatim body:
   ````
   ```
   <body, truncated to cfg.bodyTruncateBytes>
   ```
   ````
4. `## Acceptance Criteria\n\n` followed by each criterion as a
   `- ` bullet on its own line.
5. `## Constraints\n\n- Human merge required — this PR will not auto-merge.\n- Follow existing test conventions.\n- No changes outside the failing scope unless justified.`

**Truncation:** compute `Buffer.byteLength(issue.body, 'utf8')`. If
> `cfg.bodyTruncateBytes`, truncate to that many bytes using a
UTF-8-safe boundary (do not split a multi-byte char); set
`truncation.truncated = true`; record both original and truncated
byte counts.

**Acceptance criteria extraction algorithm (evaluated in order; first
non-empty wins):**
1. **Heading**: case-insensitive search for a line matching
   `/^##\s+acceptance\s+criteria\s*$/im`. Collect subsequent lines
   until next `^##` heading OR EOF. From those lines, extract
   `^\s*[-*]\s+(.+)$` bullets, trimmed. If ≥1 bullet found → use.
2. **Fenced blocks**: find `` ```test `` or `` ```expected `` fenced
   blocks (case-insensitive). For each block, take non-empty lines as
   criteria. If ≥1 criterion found → use.
3. **Synthesized fallback**: exactly one criterion:
   `Reproduce the failure described above; verify the corresponding test/check now passes after the fix.`

**Priority mapping:**
- Parse `parsePriorityLabel(issue.labels)`.
- `P0` → `'high'`; `P1` → `'high'`; `P2` → `'normal'`; `P3` → `'low'`;
  `null` → `'normal'`.

**Type mapping:**

### 1.12 `scan.ts` (TASK-012, TASK-014)

```ts
export interface SelfImproveDeps {
  config: SelfImproveConfig;
  ownership: Ownership;
  ledgerIO: LedgerIO;
  gh: GhIssueClient;
  evidence: EvidenceDeps;
  submit: SubmitDeps;
  emit: EventEmitter;
  fnRegistry: Set<string>;
  now: () => number;
  readRequestStatus?: (id: string) => Promise<'active' | 'terminal' | null>;
  readRequestCost?: (id: string) => Promise<number>;
  requestCancel?: (id: string, reason: string) => Promise<void>;
}

export function buildDefaultSelfImproveDeps(): SelfImproveDeps | null;

export interface ScanResult {
  scanned: number;
  submitted: number;
  skipped: Record<GuardId, number>;
  errors: number;
}

export function scanEnrolledRepos(
  deps: SelfImproveDeps,
  now: () => number,
): Promise<ScanResult>;
```

**Top-level flow:**
1. If `!deps.config.enabled` → emit `self_improve_disabled` (once);
   return `{ scanned: 0, submitted: 0, skipped: {}, errors: 0 }`.
2. Emit any `deps.config.configWarnings` as
   `self_improve_config_invalid`.
3. `load = loadLedger(deps.ledgerIO)`. Emit any `load.loadWarnings`
   as `self_improve_error { code: 'LEDGER_CORRUPT', ... }`.
4. Wrap the entire remaining body in `try { ... } catch (err) { emit self_improve_error; return current partial ScanResult; }`
   — NEVER propagate exceptions out of `scanEnrolledRepos`.
5. Reconciler pass (BEFORE detection):
   a. For each `entry` with `status === 'in_flight'` and
      `readRequestStatus` present, await status. `'terminal'` →
      call `readRequestCost(id)` (default 0 if absent); update
      window costs at the current hour bucket; set entry
      `status: 'idle'`, `lastOutcome` from a downstream mapping
      (`'terminal'` maps to the ledger's existing `lastOutcome`
      if set, else `'success'`). Errors here → emit
      `self_improve_error { code: 'SUBMIT_PARTIAL' }`; continue.
6. Iterate `deps.ownership.repos.filter(r => r.enrolled)` in the
   order given by ownership. For each repo:
   a. Guard `tickSubmittedSoFar < deps.config.maxIssuesPerTick`; if
      not, stop the outer loop (emit GD11 skip for remaining? — NO;
      stop silently, GD11 fires per-issue below).
   b. `list = await deps.gh.listOpen(repo.repoId, DETECTED_LABELS, 100)`;
      on error emit `self_improve_error { code: 'GH_LIST_FAILED' }`
      and continue next repo.
   c. If `list.truncated`, emit
      `self_improve_error { code: 'GH_LIST_TRUNCATED' }`.
   d. For each issue in `list.issues` (sorted by `updatedAt`
      ascending — FIFO per FR-DET-03):
      - `classification = classify(issue, { botLogin: cfg.botLogin })`.
      - If `classification.matched !== null` → emit
        `self_improve_issue_detected`.
      - Compute evidence ONLY if `classification.matched !== null`
        AND all state-only guards (`GD2`, `GD10`) would pass — but
        evidence's I/O cost is acceptable; simpler impl: always
        compute evidence when `matched !== null`.
      - Build `GuardCtx`; call `evaluateGuards(ctx)`.
      - Skip → increment `skipped[guard]`, emit
        `self_improve_issue_skipped`, continue.
      - Pass → `submit = await deps.submit.requestSubmit` via
        `submitFromIssue(...)`. On success increment `submitted`
        and `tickSubmittedSoFar`.
7. Reconcile ledger against `openIssueKeys` (all keys observed in the
   scan across all repos): `mutator.reconcile(openIssueKeys)`.
8. Orphan cancel pass (TASK-014): if `deps.requestCancel` present,
   check for `RequestSubmitResult` records in intake DB (via
   optional dep — spec deferred; see Open Questions) whose keys are
   absent from ledger; for each, call
   `requestCancel(id, 'self-improve-orphan')`, emit
   `self_improve_error { code: 'SUBMIT_PARTIAL' }`.
9. Persist ledger via `saveLedger(mutator.snapshot(), deps.ledgerIO)`.
10. Emit `self_improve_tick_summary { scanned, submitted, skipped, errors }`.

**`buildDefaultSelfImproveDeps()`:**
- Reads `readSelfImproveConfig(process.env)`. If `!enabled` → return
  `null`. Otherwise constructs production deps: audit sink from the
  same source `runWatchTick` uses; `ExecFn` from the same source
  used by `checks_client.ts`; `readOwnership()` from
  `src/ownership`.

## 3. Error Handling

### 3.1 Error taxonomy (canonical `code` values)

| Code | Thrown / Emitted by | Consumer response |
|---|---|---|
| `LOCK_BUSY` | `saveLedger` when lock unacquirable after 500 ms + stale-recovery | Emit `self_improve_error`; end tick early; return partial `ScanResult` |
| `GH_LIST_FAILED` | `gh_issues.listOpen` | Emit; skip this repo; continue |
| `GH_LIST_TRUNCATED` | `scan.ts` when `listOpen.truncated === true` | Emit; still process page-1+2 items |
| `GH_EVENTS_FAILED` | `gh_issues.getEvents` | Downgrade evidence to NA7; continue |
| `GH_COMMENT_FAILED` | `submit.ts` best-effort comment | Emit; ledger stays valid |
| `EVIDENCE_TIMEOUT` | `evidence.ts` race | Downgrade to NA7 skip |
| `SUBMIT_FAILED` | `submit.ts` when `requestSubmit` throws | Emit; ledger NOT written; next tick may retry |
| `SUBMIT_PARTIAL` | `scan.ts` orphan pass | Cancel orphan via `requestCancel` |
| `LEDGER_CORRUPT` | `loadLedger` corrupt-file preservation | Emit; in-memory starts empty |
| `CONFIG_INVALID` | `readSelfImproveConfig` (via warnings) | Emit `self_improve_config_invalid`; use defaults |
| `FN_REGISTRY_MISSING` | fn-registry loader path set + file absent | NOT an error; empty set |

### 3.2 Error classes

```ts
export class LedgerLockBusyError extends Error {
  code: 'LOCK_BUSY';
  constructor() { super('ledger lock file busy > 500 ms'); this.code = 'LOCK_BUSY'; }
}
export class LedgerKeyInvalidError extends Error {
  code: 'LEDGER_KEY_INVALID';
  constructor(key: string) { super(`invalid ledger key: ${key}`); this.code = 'LEDGER_KEY_INVALID'; }
}
```

### 3.3 User-facing messages

- `self-improve reset` unknown key: `no ledger entry for ${key}; nothing to reset` (exit 1).
- `self-improve status` corrupt: `error: ledger unreadable; corrupt file preserved at ${sidecarPath}` (exit 2).
- `self-improve tick` disabled: `self-improve disabled (set AUTONOMOUS_DEV_SELF_IMPROVE=1)` (exit 0).

### 3.4 NFR-RELIABILITY-01 — `scanEnrolledRepos` NEVER propagates

Any uncaught exception in `scanEnrolledRepos` MUST be caught and
emitted as `self_improve_error`. The function ALWAYS resolves.
`watch_tick.ts` also wraps the call in a `try/catch` as belt-and-braces
per §1.13.

## 6. Implementation Notes

### 6.1 Patterns to follow
- **Atomic write / lock file**: mirror `intake/triggers/trigger_store.ts`
  (`.tmp + rename`, mode `0o600`). Reference in code review.
- **ExecFn seam**: same as `intake/triggers/checks_client.ts`. Do NOT
  introduce a new subprocess abstraction.
- **Ownership**: reuse `src/ownership/isEnrolled` + `readOwnership`
  from existing modules. Do NOT add a new ownership model.
- **Fingerprint marker parsing**: reuse the regex style from
  `intake/triggers/issue_filer.ts` (case-insensitive `<!-- autodev-*: fp -->`).

### 6.2 Gotchas
- **`enabled` gate is strict `'1'`.** Do not accept truthy strings —
  a test in T001-03 confirms `'true'` does NOT enable the loop.
- **Guard ordering.** The 14-step order in §1.6 is
  safety-critical; keep the ordering as a verbatim comment above
  `evaluateGuards` and drive tests off a table indexed by GuardId.
- **Ledger key derivation.** ALWAYS `${repoId}#${issueNumber}` — do
  not URL-encode; `SAFE_REPO_SLUG` guarantees no problematic
  characters.
- **`readRequestStatus` absent.** Cost-window reconciliation is
  skipped silently in that case; do NOT throw.
- **Watch-tick backward-compat.** Adding `selfImproveSubmitted: number`
  MUST default to `0` at construction; existing tests read
  `result.started`, `result.reportedDone`, etc. and MUST NOT observe
  the new field unless they explicitly check for it.
- **Merge-gate is state-based, not label-based (ADR-006).** Do NOT
  key off `LABEL_SELF_FIX_PR` alone; the reference is `state.source`.
- **Body truncation is byte-accurate on UTF-8 boundaries.** Prefer
  `Buffer.from(body, 'utf8').slice(0, N).toString('utf8')` semantics
  that discard incomplete trailing bytes.

### 6.3 Similar implementations to reference
- Atomic write + lock: `plugins/autonomous-dev/intake/triggers/trigger_store.ts`.
- Data-driven catalog + iteration: `plugins/autonomous-dev/intake/triggers/issue_filer.ts`
  fingerprint marker handling.
- ExecFn wrapping: `plugins/autonomous-dev/intake/triggers/checks_client.ts`.
- Event emission via audit sink: existing usage inside
  `runWatchTick` in `plugins/autonomous-dev/intake/triggers/watch_tick.ts`.
