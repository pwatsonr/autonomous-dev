## 1. API Contracts

All contracts are bash function contracts. "Returns N" means the function exits/returns with status N. "stdout" means the function emits to standard output. All functions are expected to:

- Not perform `set -e`-violating side effects on callers that source them.
- Tolerate `set -u` (no unset-variable expansion without `${X:-}`).
- Be idempotent and have no global side effects beyond the documented stdout.

### 1.3 `verification_artifact_proof_for`

```text
NAME
    verification_artifact_proof_for

SYNOPSIS
    verification_artifact_proof_for REQ_DIR CLAIM_JSON PHASE

DESCRIPTION
    Given a request directory, an evidence-claim JSON object, and a phase
    name, search the request's phase-result artifacts[] and a bounded set
    of well-known on-disk paths under the project root for a file whose
    KIND matches the CLAIM's ecosystem, whose mtime is at-least-as-recent
    as the request's phase-dispatched timestamp, and whose content
    structurally validates as a real test/build product.

    Behaviour is gated by VERIFICATION_ARTIFACT_FALLBACK. When that env
    is "0" the function returns 1 immediately without touching the
    filesystem.

PARAMETERS
    REQ_DIR     Absolute path to the request directory
                (".../.autonomous-dev/requests/REQ-XXXXXX").
                MUST be an existing directory readable by the daemon.
    CLAIM_JSON  A single JSON object (string), the evidence row from
                phase-result-${phase}.json. MUST contain a "command"
                field (string). Other fields are optional and ignored.
    PHASE       Phase name (e.g. "integration", "deploy", "test").
                Used to locate ${REQ_DIR}/phase-result-${PHASE}.json.

OUTPUT (stdout, only on success)
    Exactly one line, the proof descriptor, in pipe-delimited form:

        <kind>:<abs_path>(|<key>=<value>)*

    Example:

        junit:/abs/project/junit.xml|tests=208|failures=0|errors=0

    KIND is one of: junit | lcov | istanbul | cobertura | vitest | jest
                    | executor_tail
    ABS_PATH is the absolute, realpath-resolved path of the artifact.
    Extras may include any of: tests, failures, errors, mtime.

RETURN
    0  Proof found and emitted on stdout.
    1  No candidate validated (no output). Includes the case
       VERIFICATION_ARTIFACT_FALLBACK=0.

EXIT SEMANTICS
    The function MUST NOT call `exit`; it MUST `return`. Callers source
    this library and depend on flow continuation.

WALL-CLOCK BUDGET
    The function MUST complete within 5 seconds wall-clock for any well-
    formed input. Enforce with an outer `timeout 5 ...` wrapper around
    the candidate-scan loop OR a deadline check inside the loop. When
    the budget expires the function returns 1 with empty stdout.

PRECONDITIONS
    - jq is on $PATH (already a hard dep of verifier.sh).
    - find is on $PATH; if absent, a hand-rolled fixed-list scan runs
      (degrades — see _artifact_candidate_paths).

POSTCONDITIONS
    - No files are created, modified, or deleted.
    - No stdout if return is 1.
    - At most one stdout line if return is 0.
```

#### Usage example

```bash
source plugins/autonomous-dev/lib/verification/artifact-proof.sh

row='{"command":"bun test","status":"pass","exit_code":0,"output_tail":""}'
if proof="$(verification_artifact_proof_for "${req_dir}" "${row}" integration)"; then
    echo "rescued by ${proof}"
else
    echo "no proof"
fi
```

### 1.4 Internal helpers (exported for tests)

#### `_artifact_candidate_paths REQ_DIR PHASE`

```text
INPUT
    REQ_DIR  request directory (absolute)
    PHASE    phase name

OUTPUT (stdout)
    Zero or more absolute paths, one per line, deduplicated, ordered:
    1. Paths from phase-result-${PHASE}.json .artifacts[] whose `kind`
       string matches the proof-kind map in §2.1 of this spec.
    2. Well-known on-disk paths under PROJECT_ROOT (resolved as
       realpath of "${REQ_DIR}/../../.."), enumerated by find with
       -maxdepth ${VERIFICATION_ARTIFACT_MAX_DEPTH} and the glob set in
       §2.2.

    Output is truncated to at most ${VERIFICATION_ARTIFACT_MAX_FILES}
    lines. Paths outside PROJECT_ROOT (after realpath) are NEVER emitted.

RETURN
    0 on success (zero or more lines).
    1 only if REQ_DIR does not exist.
```

#### `_artifact_kind_for_path PATH`

```text
INPUT   absolute or relative path
OUTPUT  one of: junit | lcov | istanbul | cobertura | vitest | jest |
        executor_tail | ""    (empty string when unknown)
RETURN  0 always (emit empty on unknown)

HEURISTICS (in order; first match wins)
    basename == "lcov.info"                                   -> lcov
    basename == "coverage-final.json"                         -> istanbul
    basename == "coverage.xml" || matches "cobertura*.xml"    -> cobertura
    basename matches "vitest*results*.json"                   -> vitest
    basename matches "jest*results*.json"                     -> jest
    basename matches "junit*.xml" || path under
        "**/surefire-reports/" || path under "**/test-results/"
                                                              -> junit
    extension == ".log"                                       -> executor_tail
    otherwise                                                 -> ""
```

#### `_artifact_ecosystem_for_command COMMAND`

```text
INPUT   shell command string (single line). First word is tokenised by
        whitespace; leading "time " / "env " / "VAR=val " prefixes are
        stripped before tokenisation.
OUTPUT  one of: js | python | rust | go | jvm | generic | ""
RETURN  0 always

MAP (case-sensitive, exact match on first token after prefix strip)
    bun|npm|pnpm|yarn|npx|node|vitest|jest|mocha|cypress|playwright -> js
    pytest|python|python3                                            -> python
    cargo                                                            -> rust
    go                                                               -> go
    gradle|gradlew|./gradlew|mvn|mvnw|./mvnw                         -> jvm
    make|bash|sh|./run|just                                          -> generic
    ""                                                               -> ""
    any other token                                                  -> ""
```

#### `_artifact_kind_matches_ecosystem KIND ECOSYSTEM`

```text
INPUT   KIND, ECOSYSTEM strings (case-sensitive)
RETURN  0 if the pair is in the allow-table; 1 otherwise.

ALLOW TABLE (an exhaustive list; every other pair is 1)
    js       -> junit, vitest, jest, istanbul, lcov, executor_tail
    python   -> junit, cobertura, executor_tail
    rust     -> junit, cobertura, executor_tail
    go       -> cobertura, executor_tail
    jvm      -> junit, cobertura, executor_tail
    generic  -> junit, vitest, jest, istanbul, lcov, cobertura, executor_tail
    ""       -> (none — always 1)

NOTES
    The "generic" ecosystem is permissive because a `make ci` / `bash run.sh`
    command may produce any runner's output.
```

#### `_artifact_is_fresh PATH REQ_DIR`

```text
INPUT   PATH (absolute path to an artifact), REQ_DIR (absolute path)
RETURN  0 if the artifact is "fresh" per the rules below; 1 otherwise.

RULES
    1. If VERIFICATION_ARTIFACT_FRESHNESS_REQUIRED=0, return 0
       unconditionally.
    2. Read REQ_DIR/state.json. Extract
         .current_phase_metadata.dispatched_at  (preferred)
       falling back to
         .updated_at
       falling back to (no dispatched_at AND no updated_at, or state.json
       missing/unreadable) -> return 0 (permissive degrade).
    3. Convert the timestamp to epoch seconds (jq -r "fromdateiso8601",
       or `date -d "..." +%s` GNU / `date -j -f "%Y-%m-%dT%H:%M:%SZ" ... +%s`
       BSD). If conversion fails -> return 0 (permissive degrade).
    4. Read the artifact's mtime in epoch seconds via _artifact_stat_mtime.
       If the stat call fails -> return 1 (file presumably gone).
    5. Return 0 iff artifact_mtime >= dispatched_epoch; else 1.
```

#### `_artifact_stat_mtime PATH`

```text
INPUT   PATH
OUTPUT  Decimal epoch seconds (integer) on stdout. Empty on failure.
RETURN  0 on success; 1 if neither stat invocation worked.

PORTABILITY
    Try `stat -c %Y PATH` first (GNU). On non-zero exit, try
    `stat -f %m PATH` (BSD/macOS). Both stderr suppressed.
```

#### `_check_artifact_<kind>` validators

All seven validators share the contract below. Differences are in the content rule.

```text
INPUT   PATH (absolute, expected readable)
OUTPUT  On success: zero or more `key=value` lines on stdout (e.g.
        "tests=208", "failures=0"). On failure: nothing.
RETURN  0 on success; 1 on failure.

COMMON PRECHECKS (applied before kind-specific rule; failure -> return 1)
    1. PATH is a readable regular file.
    2. File size (bytes) <= VERIFICATION_ARTIFACT_MAX_SIZE_MB * 1024 * 1024.
    3. The kind-specific content rule completes within 5 seconds — wrap
       any external parser call (`xmllint`, `jq`, large `grep`) in
       `timeout 5 ...`. On timeout -> return 1.

KIND RULES
    junit (_check_artifact_junit)
        - File head (first 4096 bytes) matches a regex with root
          element <testsuite ...> OR <testsuites ...>.
          Regex (POSIX ERE):
              <testsuite[s]?[[:space:]>]
        - File contains at least one <testcase ...> element
          (grep -q -E '<testcase[[:space:]/>]' over the whole file).
        - Extract `tests` attribute on the root if present:
              grep -oE 'tests="[0-9]+"' | head -1 | grep -oE '[0-9]+'
          Emit "tests=N" only when N >= 1.
        - Extract `failures` and `errors` attributes the same way; emit
          when present.

    lcov (_check_artifact_lcov)
        - File contains at least one line starting with "SF:".
        - File contains at least one line starting with "DA:".
        - No extras emitted (lcov files have no obvious summary).

    istanbul (_check_artifact_istanbul_json)
        - jq filter MUST be true:
              jq -e 'type == "object" and (to_entries
                    | map(.value | (has("path") and has("statementMap")))
                    | any)'
          (At least one entry has both `path` and `statementMap`.)
        - No extras emitted.

    cobertura (_check_artifact_cobertura_xml)
        - File head matches root <coverage ...>:
              <coverage[[:space:]>]
        - Root element contains either a `lines-covered` attribute OR a
          `line-rate` attribute. (grep -E
              '<coverage [^>]*(lines-covered|line-rate)=')
        - No extras emitted.

    vitest (_check_artifact_vitest_json)
        - jq filter MUST be true:
              jq -e '(.numTotalTests // 0) >= 1
                     and (.numTotalTestSuites // 0) >= 1'
        - Emit "tests=$(jq -r .numTotalTests)" when present.
        - Emit "failures=$(jq -r .numFailedTests)" when present.

    jest (_check_artifact_jest_json)
        Identical to vitest.

    executor_tail (_check_artifact_executor_tail)
        - Read the last 200 lines of the file (`tail -n 200`).
        - The tail MUST match at least one regex in the §2.3 set.
        - No extras emitted.
```

### 1.6 `hooks/audit-log-finalizer.sh` — PostToolUse hook (TASK-008)

```text
NAME
    audit-log-finalizer.sh

SYNOPSIS
    Invoked by the Claude SDK PostToolUse hook with a JSON event on stdin.

ENVIRONMENT
    AUDIT_LOG_PATH   Absolute path to the audit log (same as PreToolUse).
    AUDIT_PHASE      Phase name.

INPUT (stdin)
    A JSON event object. Must contain at minimum:
        .tool_name        string
        .tool_input       object (for Bash, .tool_input.command exists)
        .tool_response    object (.exit_code or .is_error, .stdout,
                          .stderr)
        .tool_use_id      string (optional in older SDKs; absence is
                          handled per below)

CONTRACT
    - If tool_name != "Bash" OR AUDIT_LOG_PATH unset/unwritable OR
      stdin is empty -> exit 0 (no-op).
    - Acquire `flock -w 1 ${AUDIT_LOG_PATH}.lock` when `flock` is on
      $PATH. On lock-acquire timeout OR flock absent: continue in
      best-effort mode (do NOT block).
    - Extract:
        tool_use_id    string  (jq -r '.tool_use_id // empty')
        command        string  (jq -r '.tool_input.command // ""')
        exit_code      int|null
                       (jq -r '.tool_response.exit_code // empty')
                       FALLBACK: if .tool_response.is_error == true,
                                 exit_code=1; if false, exit_code=0.
        duration_ms    int|null
                       (jq -r '.tool_response.duration_ms // empty')
        stdout         string  (jq -r '.tool_response.stdout // ""')
        stderr         string  (jq -r '.tool_response.stderr // ""')
        output_tail    string  last 50 lines of stdout + "\n" + stderr
                       (printf '%s\n%s' "${stdout}" "${stderr}" | tail -n 50)
    - If tool_use_id is non-empty AND a row exists in AUDIT_LOG_PATH
      with the same tool_use_id field value:
        Rewrite AUDIT_LOG_PATH via jq atomic-merge:
            tmp="${AUDIT_LOG_PATH}.tmp.$$"
            jq -c --arg id "${tool_use_id}" \
                  --argjson ec "${exit_code_or_null}" \
                  --argjson dur "${duration_ms_or_null}" \
                  --arg tail "${output_tail}" \
                  'if .tool_use_id == $id then
                      .exit_code = $ec
                      | .duration_ms = $dur
                      | .output_tail = $tail
                      | .source = (.source + "+sdk_hook_post")
                   else . end' \
                  "${AUDIT_LOG_PATH}" > "${tmp}"
            mv "${tmp}" "${AUDIT_LOG_PATH}"
        Permissions: preserve 0600 via `chmod 0600 "${tmp}"` before mv.
    - Else (no matching row, or tool_use_id absent):
        Append a fresh row with:
            ts          NOW (ISO8601 UTC w/ ms when available)
            phase       AUDIT_PHASE or "unknown"
            command     command
            argv        [first token]
            cwd         $PWD
            exit_code   exit_code_or_null
            duration_ms duration_ms_or_null
            output_tail output_tail
            tool_use_id tool_use_id (empty -> omit field via jq // empty)
            source      "sdk_hook_post"
        Append uses `printf '%s\n' "${row}" >> "${AUDIT_LOG_PATH}"`,
        wrapped in the same flock when available.
    - Always `exit 0`. The hook MUST NEVER `exit 1` or `exit 2`.

PERMISSIONS
    chmod 0755 on the hook script itself.
```

### 1.7 `hooks/audit-log-writer.sh` — Hardening (TASK-010)

Two surgical edits to the existing file (do NOT rewrite):

1. **`tool_use_id` capture.** Read `.tool_use_id // empty` from the event JSON. When non-empty, include it in the row via:

   ```bash
   tool_use_id="$(printf '%s' "${event_json}" | jq -r '.tool_use_id // empty' 2>/dev/null || true)"
   ```

   In the `jq -nc` row builder, replace the current builder with a version that conditionally adds `tool_use_id`:

   ```bash
   row="$(jq -nc \
       --arg ts "${ts}" \
       --arg phase "${phase}" \
       --arg command "${command_str}" \
       --arg first_token "${first_token}" \
       --arg cwd "${cwd}" \
       --arg tool_use_id "${tool_use_id}" \
       '{
           ts: $ts,
           phase: $phase,
           command: $command,
           argv: [$first_token],
           cwd: $cwd,
           exit_code: null,
           duration_ms: null,
           output_tail: null,
           source: "sdk_hook_pre"
       } + (if $tool_use_id == "" then {} else {tool_use_id: $tool_use_id} end)' 2>/dev/null)"
   ```

   Note the `source` value changes from `"sdk_hook"` to `"sdk_hook_pre"` to disambiguate from PostToolUse rows (which use `"sdk_hook_post"`). Existing readers (`audit_log_has_command`) match by command atoms, not by `source`, so this is non-breaking.

2. **`flock` wrapper.** Replace the final append with:

   ```bash
   if command -v flock >/dev/null 2>&1; then
       (
           flock -w 1 9 || true
           printf '%s\n' "${row}" >&9
       ) 9>> "${log_path}" 2>/dev/null || true
   else
       printf '%s\n' "${row}" >> "${log_path}" 2>/dev/null || true
   fi
   ```

   Fail-open semantics preserved.

## 2. Data Schemas

### 2.1 Recognised `artifacts[].kind` values

`phase-result-<phase>.json` already declares an `artifacts: [{kind, path, title}]` array. The fallback recognises these kind strings (case-sensitive):

| `kind` | Internal kind | Validator |
|---|---|---|
| `test-results-junit` | `junit` | `_check_artifact_junit` |
| `test-results-vitest` | `vitest` | `_check_artifact_vitest_json` |
| `test-results-jest` | `jest` | `_check_artifact_jest_json` |
| `coverage-lcov` | `lcov` | `_check_artifact_lcov` |
| `coverage-istanbul` | `istanbul` | `_check_artifact_istanbul_json` |
| `coverage-cobertura` | `cobertura` | `_check_artifact_cobertura_xml` |
| `executor-tail` | `executor_tail` | `_check_artifact_executor_tail` |

Unrecognised `kind` values are silently ignored. No schema-validator change required (existing validator accepts arbitrary kind strings).

### 2.2 Well-known on-disk path globs

Resolved relative to `PROJECT_ROOT = realpath(${req_dir}/../../..)`. Use `find` with `-maxdepth ${VERIFICATION_ARTIFACT_MAX_DEPTH}`. The globs (POSIX find name patterns):

| Glob | Maps to internal kind |
|---|---|
| `junit*.xml`, `TEST-*.xml` | `junit` |
| `coverage/lcov.info`, `lcov.info` | `lcov` |
| `coverage/coverage-final.json` | `istanbul` |
| `coverage.xml`, `cobertura.xml`, `cobertura-coverage.xml` | `cobertura` |
| `.vitest-results.json`, `vitest-results.json` | `vitest` |
| `jest-results.json` | `jest` |
| `target/surefire-reports/TEST-*.xml`, `**/surefire-reports/*.xml` | `junit` |
| `target/site/jacoco/jacoco.xml` | `cobertura` |
| `${req_dir}/test-output/*.log` | `executor_tail` |

Implementation note: enumerate via one `find PROJECT_ROOT -maxdepth N \( -name 'junit*.xml' -o -name 'lcov.info' -o ... \) -type f` invocation, then post-filter for paths still inside `realpath(PROJECT_ROOT)` (no symlink escape).

If `find` is absent, fall back to a fixed list of literal paths (just the basename hits at PROJECT_ROOT and one level down, no recursion).

### 2.3 Executor-tail regex set

`_check_artifact_executor_tail` accepts a file whose last 200 lines match **any** of these POSIX ERE regexes (matched line-by-line via `grep -E`):

| ID | Regex | Runner |
|---|---|---|
| ET-1 | `Tests:[[:space:]]+[0-9]+[[:space:]]+passed` | Jest, Vitest |
| ET-2 | `[0-9]+[[:space:]]+passing\b` | Mocha |
| ET-3 | `[0-9]+[[:space:]]+passed,[[:space:]]+[0-9]+[[:space:]]+failed` | pytest |
| ET-4 | `test result: ok\.[[:space:]]+[0-9]+[[:space:]]+passed` | cargo test |
| ET-5 | `ok[[:space:]]+[0-9]+ - ` (and a `1\.\.[0-9]+` plan line elsewhere in tail) | bats / TAP |
| ET-6 | `^PASS[[:space:]]` (must match ≥3 distinct lines) | Go `go test` |
| ET-7 | `^BUILD SUCCESS\b` | Maven |
| ET-8 | `BUILD SUCCESSFUL in [0-9]+s` | Gradle |
| ET-9 | `[0-9]+[[:space:]]+tests?[[:space:]]+complete[d]?` | generic |

Implementation: bash array of `(id, regex)` pairs; iterate; on any match return 0 immediately. For ET-5 BOTH conditions must hold (the `ok N - ` line AND a separate `1..N` plan line within the tail). For ET-6 require ≥3 lines (`grep -cE` >= 3).

| Field | Type | Required | Source |
|---|---|---|---|

| `tests` | integer | optional | extras `tests=N` |
| `failures` | integer | optional | extras `failures=N` |

```bash
| (if $proof_json == "" then {} else {artifact_proof: ($proof_json | fromjson)} end)
```

## 3. Error Handling

### 3.1 Error taxonomy (artifact-proof.sh)

| Failure mode | Detection point | Recovery |
|---|---|---|
| `REQ_DIR` does not exist | `_artifact_candidate_paths` | Return 1 with empty stdout. |
| `phase-result-${phase}.json` missing/unparseable | `_artifact_candidate_paths` step 1 | Continue with step 2 (well-known paths only). |
| `find` not installed | step 2 enumeration | Fallback to fixed-list scan. Emit only literal-path hits. |
| `realpath` not available | path resolution | Use `cd -P "${REQ_DIR}/../../.." && pwd`; if that fails, skip the well-known-path scan entirely. |
| Artifact file unreadable | per-validator precheck | Validator returns 1. Continue to next candidate. |
| Artifact > size cap | per-validator precheck | Validator returns 1. |
| Validator parser hangs | 5s `timeout` wrapper | Validator returns 1. |
| `jq` missing | any jq shell-out | Validator returns 1 (treat as not-a-proof). Verifier already requires jq, so this is a degraded-runtime edge case. |
| `state.json` missing | freshness gate | Return 0 from `_artifact_is_fresh` (permissive degrade). |
| `state.json` lacks `dispatched_at` AND `updated_at` | freshness gate | Return 0 (permissive degrade). |
| Per-claim wall-clock exceeds 5s | outer guard in `verification_artifact_proof_for` | Return 1 with empty stdout. |

### 3.2 Error taxonomy (audit-log-finalizer.sh)

| Failure mode | Detection point | Recovery |
|---|---|---|
| `tool_name != "Bash"` | stdin parse | Exit 0 (no-op). |
| `AUDIT_LOG_PATH` unset | env check | Exit 0 (no-op). |
| Stdin empty | event read | Exit 0 (no-op). |
| `jq` missing / event not JSON | jq parse | Exit 0 (no-op). |
| `flock` not on $PATH | lock acquire | Skip locking; continue best-effort append/rewrite. |
| `flock` timeout (1s) | lock acquire | Continue best-effort (matches PreToolUse hook contract). |
| Atomic rewrite fails (mv error) | merge path | Discard `${tmp}` (rm -f), exit 0. The PreToolUse row remains in place. |
| `tool_use_id` absent in SDK event | jq extract | Append as backfill row (no merge attempted). |

All hook code paths MUST end in `exit 0`. The hook MUST NEVER block tool execution.

### 3.3 Error taxonomy (verifier.sh branch)

| Failure mode | Detection | Recovery |
|---|---|---|
| `artifact-proof.sh` missing at source-time | shell `.` builtin fails | The verifier MUST still run. Use `. "${__VERIFIER_DIR__}/artifact-proof.sh" 2>/dev/null || true` AND guard the fallback branch with `if declare -F verification_artifact_proof_for >/dev/null 2>&1; then ...`. |
| `verification_artifact_proof_for` returns non-zero | call site | Continue with original `verdict="would_have_failed"` path (no behaviour change). |
| Proof descriptor malformed | post-call parse | Treat as no rescue: keep `would_have_failed`. Log to stderr at WARN: `verifier: malformed artifact_proof descriptor: <descriptor>`. |
| Unknown verdict in counter case | counter switch | The default branch MUST emit `echo "verifier: unknown verdict ${verdict}" >&2` and treat as `would_have_failed`. |

## 5. Implementation Notes

### 5.1 Patterns to follow

- **Source guards.** Match `verifier.sh`'s use of `__VERIFIER_DIR__`. Use a `__ARTIFACT_PROOF_SH_SOURCED__` guard at the top of `artifact-proof.sh`.
- **JSONL row composition.** Follow the `jq -nc --arg ... '{...}'` pattern from the existing `audit-log-writer.sh` lines 89–105.
- **Atomic rewrite.** Use `tmp="${path}.tmp.$$"` then `mv "${tmp}" "${path}"`. Preserve mode 0600 via `chmod` on the tmp before mv (matches `audit_log_file` creation in `spawn-session.sh` L279).
- **Fail-open hook contract.** Mirror `audit-log-writer.sh`'s exit-0-always semantics. Every code path in `audit-log-finalizer.sh` must reach `exit 0`.
- **Project root resolution.** Match the verifier's existing pattern at L585–L591 (`cd "${req_dir}/../../.." 2>/dev/null && pwd`). Use the same in `_artifact_candidate_paths`.

### 5.2 Known gotchas

- **BSD vs GNU `find`.** Avoid `-printf`, `-quit`, `-regextype`. Stick to `-name`, `-type f`, `-maxdepth`. macOS BSD `find` and GNU `find` both support these.
- **BSD vs GNU `stat`.** GNU is `stat -c %Y`. BSD/macOS is `stat -f %m`. Implement `_artifact_stat_mtime` to try GNU first, fallback to BSD.
- **BSD vs GNU `date`.** For mtime → ISO8601 conversion, try GNU `date -u -d @<epoch> +"%Y-%m-%dT%H:%M:%SZ"` first; fallback to BSD `date -u -r <epoch> +"%Y-%m-%dT%H:%M:%SZ"`.
- **`touch` for fixture backdating.** GNU supports `touch -d "1 day ago"`; BSD supports `touch -t YYYYMMDDhhmm`. Tests that backdate artifacts (AF-06) should detect the platform once and use the right form, e.g.:

  ```bash
  if touch -d "1 day ago" "${tmp}/probe" 2>/dev/null; then
      backdate() { touch -d "$1" "$2"; }
  else
      backdate() { touch -t "$(date -u -v-1d +%Y%m%d%H%M)" "$2"; }
  fi
  ```

- **`flock` graceful absence.** macOS does not ship `flock` by default. All `flock` usage MUST be gated by `command -v flock >/dev/null 2>&1`. The race test in AF-FIN-05 MUST `skip` when `flock` is absent.
- **`jq` is hard-required.** `verifier.sh` already requires `jq`. Do NOT add a `jq`-absent fallback in `artifact-proof.sh` — if `jq` is missing, the verifier itself is broken upstream and the validator returning 1 is the correct degraded behaviour.
- **Settings JSON line-noise in `spawn-session.sh`.** The current `jq -n` pretty-prints to a temp file. Keep the same `jq -n` invocation pattern and the same temp-file `trap` cleanup pattern; just extend the JSON object.
- **`source` field rename.** `audit-log-writer.sh` currently emits `source: "sdk_hook"`. The hardening (TASK-010) changes this to `"sdk_hook_pre"`. Update any tests that hard-code the literal `"sdk_hook"` string. Run `grep -rn '"sdk_hook"' plugins/autonomous-dev/tests` during TASK-010 and update each match to `"sdk_hook_pre"` OR loosen to `^sdk_hook(_pre)?$` regex.

- **JSONL append.** `plugins/autonomous-dev/hooks/audit-log-writer.sh` L86–L112.
