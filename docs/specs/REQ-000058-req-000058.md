## 1. API Contracts

### 1.1 `daemon_run_tests` (bash function)

Location: `plugins/autonomous-dev/lib/verification/daemon-run-tests.sh`.

#### 1.1.1 Signature

```bash
# daemon_run_tests <req_dir> <project_root> <phase>
#
# Positional args:
#   $1 req_dir       — absolute path to <project>/.autonomous-dev/requests/<request_id>
#                      (directory must exist; MUST be writable by the daemon uid).
#   $2 project_root  — absolute path to the repo root (contains package.json /
#                      lockfile). MUST be a directory. Used as CWD of the runner.
#   $3 phase         — one of "integration" | "test" | "deploy". Written into
#                      the artefact's .phase field. Any other value is accepted
#                      (defensive) but caller SHOULD gate.
#
# Preconditions:
#   - jq (>=1.6) on PATH.
#   - GNU or BSD `date`, `stat` on PATH (function must self-detect).
#   - Optional: `resolve_timeout_bin` (from supervisor-loop.sh) present in
#     the caller's shell. If unavailable, fall back to no-timeout invocation
#     and log an event (see §1.1.7).
#
# Return codes:
#   0  → artefact written to <req_dir>/test-results.json with .status="pass"
#        OR .status="fail" (i.e. the run completed — caller inspects .failures)
#   1  → no detectable runner (no package.json OR scripts.test unresolvable
#        AND no node_modules/.bin/{vitest,jest}); NO artefact written.
#   2  → infrastructure error, timeout, write error, or reporter parse failure;
#        artefact written with .status in { "timeout" | "infrastructure_error" }.
#
# Side effects:
#   - Writes <req_dir>/test-results.json (mode 0600, single JSON object).
#   - Writes <req_dir>/test-output/daemon-rerun.log (mode 0600, tail-truncated
#     to <= 256 KiB of stdout+stderr; older content dropped at head).
#   - Appends events to <req_dir>/events.jsonl (mode 0600):
#       * daemon_test_rerun_started
#       * daemon_test_rerun_completed
#       * daemon_test_rerun_skipped_no_runner
#       * daemon_test_rerun_timeout
#       * daemon_test_rerun_write_failed
#       * daemon_test_rerun_no_tests_collected (§1.1.5 vacuous-pass rule)
#
# Non-side-effects:
#   - MUST NOT mutate git state, npm cache, or files under project_root
#     other than what the runner itself touches.
#   - MUST NOT exit the caller's shell (no `exit`; only `return`).
daemon_run_tests() { ... }
```

#### 1.1.2 Kill-switch semantics

The function MUST honour `VERIFICATION_DAEMON_RERUN` **before any work**:

```bash
daemon_run_tests() {
    if [[ "${VERIFICATION_DAEMON_RERUN:-1}" != "1" ]]; then
        _daemon_emit_event "${req_dir}" "daemon_test_rerun_skipped_kill_switch" \
            "$(jq -nc --arg p "${phase}" '{phase:$p, reason:"kill_switch"}')" || true
        return 1
    fi
    # ...
}
```

Kill-switch return code is **1** (no artefact) so verifier's presence-miss rescue falls through to the existing artifact-proof path (no regression).

#### 1.1.3 Runner detection (`_daemon_detect_runner`)

Internal helper.

```bash
# _daemon_detect_runner <project_root>
# Writes a jq-compatible JSON object to stdout, one line, e.g.:
#   {"runner":"vitest","cmd_prefix":"npx vitest run","raw":".scripts.test"}
# Returns 0 always. Callers inspect the .runner field.
```

Detection priority (first match wins):

| # | Condition | Result |
|---|-----------|--------|
| 1 | `VERIFICATION_DAEMON_RERUN_RUNNER` env is `vitest`/`jest`/`npm_script` | `runner=<that value>` |
| 2 | `[[ ! -f "${project_root}/package.json" ]]` | `runner="none"`, `reason="no_package_json"` |
| 3 | `jq -r '.scripts.test // empty' package.json` is non-empty | `runner="npm_script"`, `raw="<script>"` |
| 4 | `[[ -x "${project_root}/node_modules/.bin/vitest" ]]` | `runner="vitest"` |
| 5 | `[[ -x "${project_root}/node_modules/.bin/jest" ]]` | `runner="jest"` |
| 6 | otherwise | `runner="none"`, `reason="no_runner_detected"` |

**npm_script sub-classification.** When `runner="npm_script"`, examine `raw` (case-insensitive substring, in order):

- If `raw` matches `\bvitest\b` → sub-runner `vitest`, JSON-reporter wrapping = vitest form (§1.1.4).
- Else if `raw` matches `\bjest\b` → sub-runner `jest`, JSON-reporter wrapping = jest form.
- Else → sub-runner `unknown`, fall back to tail-parse (§1.1.5).

Substring detection is a heuristic — no shell evaluation of `raw`.

#### 1.1.4 JSON reporter wrapping

Table (verbatim per TDD §4.1.2 — do not "improve" flags):

| Runner (or npm_script sub-runner) | Invoked command (argv-array form) |
|-----------------------------------|-----------------------------------|
| `vitest` (direct) | `( "npx" "vitest" "run" "--reporter=json" "--outputFile=${req_dir}/test-results.json" )` |
| `jest` (direct) | `( "npx" "jest" "--ci" "--json" "--outputFile=${req_dir}/test-results.json" )` |
| `npm_script`, sub-runner `vitest` | `( "npm" "test" "--" "--reporter=json" "--outputFile=${req_dir}/test-results.json" )` |
| `npm_script`, sub-runner `jest` | `( "npm" "test" "--" "--json" "--outputFile=${req_dir}/test-results.json" )` |
| `npm_script`, sub-runner `unknown` | `( "npm" "test" )` — no reporter wrapping; tail-parse path. |

**Argv array (NOT `bash -c "$cmd"`).** The runner MUST be invoked via an
argv array to prevent shell interpolation of `scripts.test` contents (TDD §7).
Reference pattern:

```bash
local -a inv=( "npm" "test" "--" "--reporter=json" "--outputFile=${req_dir}/test-results.json" )
"${timeout_bin}" "${timeout_s}s" "${inv[@]}"
```

When the JSON reporter succeeds and produces valid JSON at
`<req_dir>/test-results.json`, the function overwrites that file with the
**normalised schema** (§2 below) — the runner-produced JSON is transient and
does NOT match our schema.

#### 1.1.5 Tail-parse fallback

Applies when runner produces no native JSON (unknown npm_script) OR JSON reporter output is malformed. Regex family MUST literal-copy from `plugins/autonomous-dev/lib/verification/artifact-proof.sh:534–598` (function `_check_artifact_executor_tail`). Match order:

| Priority | Pattern (extended regex) | Extract |
|----------|--------------------------|---------|
| ET-1 (vitest/jest) | `Tests:[[:space:]]+([0-9]+)[[:space:]]+passed` (with optional `([0-9]+)[[:space:]]+failed`) | `passed`, `failed` |
| ET-3 (pytest) | `([0-9]+)[[:space:]]+passed,[[:space:]]*([0-9]+)[[:space:]]+failed` OR `([0-9]+)[[:space:]]+passed(,|[[:space:]])` | `passed`, `failed` (if absent → 0) |
| ET-4 (cargo) | `test result: ok\.[[:space:]]+([0-9]+)[[:space:]]+passed` | `passed`, `failed=0` |
| ET-2 (mocha) | `([0-9]+)[[:space:]]+passing` | `passed`, `failed=0` |
| Fallback | No pattern matched | `tests=0`, `passed=0`, `failed=0`, `.source="tail_parsed"` — vacuous pass; MUST emit `daemon_test_rerun_no_tests_collected`. |

Tail source: last 200 lines of the runner's combined stdout+stderr (mirrors artifact-proof.sh's `tail -n 200`).

#### 1.1.6 Working directory, env, timeout

Per TDD §4.1.4:

- **CWD.** `cd "${project_root}"` before invoking runner. Never run in `req_dir`.
- **Env allow-list.** Exported to child: `PATH`, `HOME`, `USER`, `CI=1`, `LANG`, `LC_ALL`, `TMPDIR`. All other env is either untouched (inherited from parent) or scrubbed as necessary; **do NOT export `NODE_OPTIONS`** into the child (may inject debuggers). Explicit: `unset NODE_OPTIONS` before argv-array invocation.
- **Timeout.** Read `VERIFICATION_DAEMON_RERUN_TIMEOUT_S`, default `${VERIFICATION_REEXEC_TIMEOUT_S:-300}`. If `declare -F resolve_timeout_bin` is available call it; else use `command -v gtimeout || command -v timeout || echo ""` — empty string means no timeout wrapping (log a `daemon_test_rerun_no_timeout_bin` event and proceed).
- **Timeout exit code 124** → write artefact with `status="timeout"`, `exit_code=124`, emit `daemon_test_rerun_timeout`, return 2.

#### 1.1.7 Idempotency, ownership, concurrency

- Overwrites `<req_dir>/test-results.json` on re-invocation (per TDD §5.1). Callers MUST NOT depend on preservation.
- Artefact + tail log written 0600, owned by daemon uid. Explicit `chmod 600` after write.
- Not designed for concurrent invocation for the same `(req_dir, phase)`. No lockfile; supervisor guarantees single agent per `(request_id, phase)`.

#### 1.1.8 Usage example

```bash
# From supervisor-loop.sh::dispatch_phase, after belt-and-suspenders synth.
source "${PLUGIN_DIR}/lib/verification/daemon-run-tests.sh"
if [[ "${VERIFICATION_DAEMON_RERUN:-1}" == "1" ]] \
   && [[ "${phase}" == "integration" || "${phase}" == "test" ]] \
   && declare -F daemon_run_tests >/dev/null 2>&1; then
    daemon_run_tests "${req_dir}" "${project}" "${phase}" \
        >> "${output_file:-/dev/null}.daemon-rerun" 2>&1 || true
fi
```

### 1.2 `_verifier_daemon_rerun_confirms` (bash helper)

Location: `plugins/autonomous-dev/lib/verification/verifier.sh` (co-located with `classify_command` / `audit_log_has_command`).

#### 1.2.1 Signature

```bash
# _verifier_daemon_rerun_confirms <req_dir> <phase_started_at_iso>
#
# Args:
#   $1 req_dir              — absolute path (must be a directory)
#   $2 phase_started_at_iso — ISO-8601 UTC string (e.g. "2026-07-01T21:14:10Z");
#                             MAY be empty ("") — in which case freshness
#                             check is SKIPPED (still requires .status=pass).
#
# Returns:
#   0 iff ALL of:
#       - "${req_dir}/test-results.json" exists and is a regular file
#       - jq '.schema_version == 1' matches
#       - jq '.status == "pass"' matches
#       - jq '.failures == 0' matches
#       - artefact mtime >= parse_iso($2) when $2 non-empty
#   1 otherwise (missing file, non-regular, jq parse error, schema mismatch,
#     non-pass status, non-zero failures, stale mtime)
#
# MUST NOT abort verifier on any error path. All internal jq calls MUST
# tolerate malformed JSON (route to return 1).
```

#### 1.2.2 Reference implementation shape

```bash
_verifier_daemon_rerun_confirms() {
    local req_dir="$1" phase_started_at="${2:-}"
    local art="${req_dir}/test-results.json"
    [[ -f "${art}" ]] || return 1

    local schema status failures
    schema=$(jq -r '.schema_version // empty' "${art}" 2>/dev/null) || return 1
    [[ "${schema}" == "1" ]] || return 1
    status=$(jq -r '.status // empty' "${art}" 2>/dev/null) || return 1
    [[ "${status}" == "pass" ]] || return 1
    failures=$(jq -r '.failures // 1' "${art}" 2>/dev/null) || return 1
    [[ "${failures}" == "0" ]] || return 1

    if [[ -n "${phase_started_at}" ]]; then
        local mtime phase_unix
        mtime=$(stat -f '%m' "${art}" 2>/dev/null || stat -c '%Y' "${art}" 2>/dev/null || echo 0)
        phase_unix=$(date -j -f "%Y-%m-%dT%H:%M:%SZ" "${phase_started_at}" +%s 2>/dev/null \
            || date -d "${phase_started_at}" +%s 2>/dev/null || echo 0)
        [[ "${mtime}" -ge "${phase_unix}" ]] || return 1
    fi
    return 0
}
```

### 1.3 `_is_test_runner_command` (bash helper)

Location: `plugins/autonomous-dev/lib/verification/verifier.sh`, alongside `classify_command`.

#### 1.3.1 Signature

```bash
# _is_test_runner_command <cmd>
#
# Args:
#   $1 cmd — a single-line shell command string (may include args & flags,
#            already unquoted; caller's normalization: leading/trailing ws
#            trimmed).
#
# Returns:
#   0 iff cmd's *leading token(s)* match one of the following test-runner
#     patterns (anchored at start, case-sensitive):
#
#       bun[[:space:]]+test(\b|$)
#       npm[[:space:]]+test(\b|$)
#       pnpm[[:space:]]+test(\b|$)
#       yarn[[:space:]]+test(\b|$)
#       npx[[:space:]]+vitest(\b|$)
#       npx[[:space:]]+jest(\b|$)
#       vitest(\b|$)
#       jest(\b|$)
#       pytest(\b|$)
#       python[[:space:]]+-m[[:space:]]+pytest(\b|$)
#       python3[[:space:]]+-m[[:space:]]+pytest(\b|$)
#       cargo[[:space:]]+test(\b|$)
#       go[[:space:]]+test(\b|$)
#
#   1 otherwise. Explicitly excludes:
#       git *, gh *, npm publish, npm run <anything-but-test>, docker *,
#       kubectl *, aws *, terraform *, rm *, mv *, cp *, curl *, wget *
#
# MUST use anchored/leading-token match. A substring anywhere in the string
# (e.g. `echo "run npm test"`) MUST NOT match.
```

#### 1.3.2 Reference implementation shape

```bash
_is_test_runner_command() {
    local cmd="${1:-}"
    # Strip leading whitespace only; do NOT interpret the command.
    cmd="${cmd#"${cmd%%[![:space:]]*}"}"
    case "${cmd}" in
        "bun test"|"bun test "*)                return 0 ;;
        "npm test"|"npm test "*)                return 0 ;;
        "pnpm test"|"pnpm test "*)              return 0 ;;
        "yarn test"|"yarn test "*)              return 0 ;;
        "npx vitest"|"npx vitest "*)            return 0 ;;
        "npx jest"|"npx jest "*)                return 0 ;;
        "vitest"|"vitest "*)                    return 0 ;;
        "jest"|"jest "*)                        return 0 ;;
        "pytest"|"pytest "*)                    return 0 ;;
        "python -m pytest"|"python -m pytest "*)   return 0 ;;
        "python3 -m pytest"|"python3 -m pytest "*) return 0 ;;
        "cargo test"|"cargo test "*)            return 0 ;;
        "go test"|"go test "*)                  return 0 ;;
    esac
    return 1
}
```

#### 2.1.1 Status derivation rules

| exit_code | failures (post-parse) | source | Resulting status |
|-----------|-----------------------|--------|------------------|
| 0 | 0 | `native_json` or `tail_parsed` | `pass` |
| 0 | >0 | any | `fail` |
| nonzero (not 124) | 0 | `native_json` | `pass` (trust reporter; TDD §6) |
| nonzero (not 124) | 0 | `tail_parsed` | `fail` |
| nonzero (not 124) | >0 | any | `fail` |
| 124 | any | any | `timeout` |
| runner not found / exec fail | n/a | `infrastructure_error` | `infrastructure_error` |

Verdict rule for verifier consumption (TDD §4.3.2): **only `status="pass" ∧ failures=0` counts as proof.** `timeout`, `infrastructure_error`, and `fail` are all NOT proof.

```bash
# REQ-000058: prefer the daemon's canonical re-execution artefact.

if _verifier_daemon_rerun_confirms "${req_dir}" "${phase_started_at:-}"; then
    verdict="verified_by_artifact"
    reason="artifact_proof=daemon_rerun:${req_dir}/test-results.json"

    _art_proof="$(verification_artifact_proof_for \
        "${req_dir}" "${row}" "${phase}" 2>/dev/null || true)"
    if [[ -n "${_art_proof}" ]]; then
        verdict="verified_by_artifact"
        reason="artifact_proof=${_art_proof}"
    fi
fi
```

## 7. Error Handling

### 7.1 Error taxonomy (this REQ)

| Origin | Error | Return | Artefact `status` | Verifier verdict |
|--------|-------|--------|-------------------|------------------|
| Detector | No package.json | 1 | (none) | rescue falls through to existing artifact-proof; then existing verifier verdict |
| Detector | scripts.test empty & no runner | 1 | (none) | same as above |
| Reporter | Runner not installed | 2 | `infrastructure_error` | NOT proof |
| Reporter | Runner exit=1, failures=1 | 0 | `fail` | NOT proof |
| Reporter | Runner exit=1, failures=0, source=`native_json` | 0 | `pass` | proof (trust reporter) |
| Reporter | Runner exit=1, failures=0, source=`tail_parsed` | 0 | `fail` | NOT proof |
| Reporter | Runner timeout | 2 | `timeout` | NOT proof |
| I/O | test-results.json write failed | 2 | (partial) | NOT proof (file may not exist) |
| Verifier helper | jq parse failure on artefact | (helper `1`) | n/a | rescue falls through |
| Verifier helper | Malformed / non-schema-v1 artefact | (helper `1`) | n/a | rescue falls through |

### 7.2 Recovery strategy summary

- **Producer failures NEVER cascade to phase failure.** All producer invocation sites use `|| true`.
- **Consumer helpers NEVER abort verifier.** All jq calls wrapped `2>/dev/null || return 1`.
- Verifier retains prior behaviour (existing artifact-proof fallback) whenever the new daemon-rerun signal is absent, malformed, or negative.
