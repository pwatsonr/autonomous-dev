### A1. `escalate_to_paused` — Modified Function (TASK-001)

**Existing signature (UNCHANGED):**

```bash
# escalate_to_paused(request_id: string, project: string, phase: string, retry_count: int) -> void
escalate_to_paused() { ... }
```

| Parameter | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `$1 request_id` | string | non-empty; matches `REQ-\d{6}` in practice | Request to pause |
| `$2 project` | string | absolute path to a project dir containing `.autonomous-dev/requests/<request_id>/state.json` | Project root |
| `$3 phase` | string | non-empty; one of the configured phase names (`intake`, `prd`, …, `monitor`) | Phase that exhausted retries |
| `$4 retry_count` | integer | >= 0; JSON-safe (passed to `jq --argjson`) | Retry count at exhaustion |

**Return:** void (exit status 0 on all paths). Function is non-fatal; ledger failures must not abort the pause.

**Preconditions:**
- `state.json` exists and is valid JSON readable by `jq`.
- `events.jsonl` exists or is creatable in the same dir.
- `sqlite3` is *not* a precondition — its absence is handled by `sync_intake_db_row` (logs ERROR, returns 0).

**Postconditions (POST-FIX, MUST hold simultaneously):**
1. `state.json.status == "paused"` (unchanged behaviour).
2. `state.json.updated_at == "$ts"` where `$ts` is the function's local timestamp (unchanged behaviour).
3. `state.json.current_phase_metadata.paused_reason == "Retry exhaustion in phase ${phase}"` (unchanged behaviour).
4. `state.json.current_phase_metadata.paused_at == "$ts"` (unchanged behaviour).
5. One `retry_exhaustion` event appended to `events.jsonl` (unchanged behaviour).
6. **NEW:** Exactly one invocation of `sync_intake_db_row "${request_id}" "${current_phase}" "paused" "${ts}"` after the `mv` and after the `events.jsonl` append, before `emit_alert`/`log_error`.
7. **NEW:** `${current_phase}` MUST be sourced from `jq -r '.current_phase // empty' "${state_file}"`. If that read yields empty (e.g. malformed/empty state.json), the caller-supplied `$3 phase` argument MUST be used as fallback. The variable passed to the helper MUST be non-empty.
8. `emit_alert` and `log_error` calls execute (unchanged behaviour).
9. If `sync_intake_db_row` writes successfully, `intake.db.requests` row for `request_id` has `status='paused'`, `current_phase=<current_phase>`, `updated_at=<ts>`.
10. If the ledger write fails, `state.json` and `events.jsonl` are still updated; the function still returns 0.

**Side effects:**
- Atomic write of `state.json` via `tmp + mv`.
- Append to `events.jsonl`.
- One additional `sqlite3` process invocation (~1ms, see NFR-2 in TDD).
- One additional log line (INFO on success, ERROR/WARN on failure paths).
- `emit_alert` (unchanged).

**Errors:** none thrown. All failure modes inside `sync_intake_db_row` are logged and absorbed (see §Error Handling).

**Exact Pseudocode (authoritative — matches TDD §4.1, expand the existing function in place):**

```bash
escalate_to_paused() {
    local request_id="$1"
    local project="$2"
    local phase="$3"
    local retry_count="$4"
    local state_file="${project}/.autonomous-dev/requests/${request_id}/state.json"
    local events_file="${project}/.autonomous-dev/requests/${request_id}/events.jsonl"

    local ts
    ts=$(date -u +"%Y-%m-%dT%H:%M:%SZ")

    # --- UNCHANGED: atomic write of paused state ---
    local tmp="${state_file}.tmp"
    jq --arg ts "${ts}" \
        --arg reason "Retry exhaustion in phase ${phase}" \
        '
        .status = "paused" |
        .current_phase_metadata.paused_reason = $reason |
        .current_phase_metadata.paused_at = $ts |
        .updated_at = $ts
        ' "${state_file}" > "${tmp}"
    mv "${tmp}" "${state_file}"

    # --- NEW (REQ-000014 FR-A1, ADR-3): read current_phase for ledger mirror ---
    local current_phase
    current_phase=$(jq -r '.current_phase // empty' "${state_file}" 2>/dev/null || echo "")
    if [[ -z "${current_phase}" ]]; then
        current_phase="${phase}"
    fi

    # --- UNCHANGED: events.jsonl append ---
    local event
    event=$(jq -n \
        --arg ts "${ts}" \
        --arg req "${request_id}" \
        --arg phase "${phase}" \
        --argjson retries "${retry_count}" \
        '{
            timestamp: $ts,
            type: "retry_exhaustion",
            request_id: $req,
            details: {
                phase: $phase,
                retry_count: $retries,
                escalated_to: "paused"
            }
        }')
    echo "${event}" >> "${events_file}"

    # --- NEW (REQ-000014 FR-A1, FR-A2, FR-A3): mirror to intake.db. Non-fatal. ---
    sync_intake_db_row "${request_id}" "${current_phase}" "paused" "${ts}"

    # --- UNCHANGED: alert + log ---
    emit_alert "retry_exhaustion" "Request ${request_id} paused after ${retry_count} retries in phase '${phase}'"
    log_error "Request ${request_id} escalated to PAUSED (retry exhaustion in ${phase})"
}
```

## Error Handling

This spec introduces no new error types. All failure modes are inherited from `sync_intake_db_row` (REQ-000013). The taxonomy is reproduced here for completeness.

### Error Taxonomy

| Failure | Detection | Log Level | Log Format (verify against helper source) | Effect on `escalate_to_paused` |
|---------|-----------|-----------|-------------------------------------------|--------------------------------|
| `sqlite3` not on `PATH` | `command -v sqlite3` | ERROR | `sync_intake_db_row: sqlite3 CLI not found; cannot mirror ${request_id} -> ${phase}` | Pause completes; `state.json` is paused; ledger untouched |
| `INTAKE_DB` file missing | `[[ ! -f "$INTAKE_DB" ]]` | WARN | `sync_intake_db_row: ${INTAKE_DB} not present; skipping ledger update for ${request_id}` | Pause completes; ledger untouched |
| Unmapped status (cannot occur for `paused`) | `map_state_status_to_intake` returns 1 / empty | ERROR | `sync_intake_db_row: unmapped state status '${state_status}' for ${request_id}; not writing ledger` | Pause completes; ledger untouched |
| `sqlite3` non-zero exit (corrupt DB, lock failure, syntax error) | `rc -ne 0` after heredoc | ERROR | `sync_intake_db_row: sqlite3 UPDATE failed for ${request_id} -> (${phase}, ${mapped}); rc=${rc}; out=${update_out}` | Pause completes; ledger untouched |
| Zero rows matched (no ledger row for request) | `changes()` == 0 | WARN | `sync_intake_db_row: 0 rows updated for ${request_id} (no ledger row?) -> (${phase}, ${mapped})` | Pause completes; ledger unchanged |
| Success | `changes()` > 0 | INFO | `sync_intake_db_row: ledger updated ${request_id} -> (${phase}, ${mapped})` | Pause completes; ledger row reflects new phase/status |

### Recovery Strategies

- **Caller (`escalate_to_paused`):** No recovery. The helper's "always returns 0" contract means there is nothing to catch. The pause is authoritative in `state.json`.
- **Operator:** On ERROR/WARN log lines, operator inspects log and may run the orphan reconciliation path manually to re-sync (out of scope for this spec).
- **No retries.** A single best-effort attempt; the orphan-reconciler is the catch-all.

### Categorisation

- All errors are **recoverable** (non-fatal): they degrade ledger freshness but never block the pause.
- No exceptions are thrown; no Bash `set -e` violations.
- `escalate_to_paused` MUST NOT add any error-handling code around the `sync_intake_db_row` call. The call is bare:
  ```bash
  sync_intake_db_row "${request_id}" "${current_phase}" "paused" "${ts}"
  ```
  No `|| true`, no `if !`, no captured rc. The helper guarantees rc=0.

## Implementation Notes

### Patterns to Follow

- **Bats setup idiom:** Copy from `plugins/autonomous-dev/tests/bats/sync_intake_db_row.bats:1–37`. The HOME-before-source pattern is mandatory because `INTAKE_DB`, `DAEMON_HOME`, `LOG_DIR`, `LOG_FILE` are all readonly derivatives.
- **Function-call sourcing:** Tests source `supervisor-loop.sh`; do NOT execute it as a subprocess (would trigger the main loop). The `set +e ... source ... set -e` pattern guards against the helper's early `set -e` triggering on benign branches during source.
- **jq filter byte-stability:** When modifying `escalate_to_paused`, preserve the existing jq pipeline character-for-character. Use a diff tool to confirm before committing.

### Known Gotchas

- **G1: Bash `set -euo pipefail` in `supervisor-loop.sh`.** The new `current_phase=$(jq ... || echo "")` MUST keep the `|| echo ""` fallback to avoid errexit firing on a missing/empty `.current_phase`. Verified safe in TDD §4.1 pseudocode.
- **G2: Line-number drift.** TASK-001's region is ~line 1678, TASK-002's region is ~line 2250. If unrelated edits land between TDD and SPEC implementation, the line numbers above shift. The executor MUST locate by function name (`escalate_to_paused`) and by the `mv "${tmp}" "${state_file}"` anchor — never trust absolute line numbers.
- **G3: sqlite3 PATH hash.** When tests manipulate `PATH` to inject a shim, `hash -r` is required because bash caches command lookups. See REQ-000013 test U7 for the precedent.
- **G4: Schema location.** Schema lives at `plugins/autonomous-dev/intake/db/schema.sql` (confirmed). Use `$PLUGIN_DIR_PATH/intake/db/schema.sql` exactly as in `sync_intake_db_row.bats`.
- **G5: Log file rotation.** `$LOG_FILE` may accumulate from previous tests in the same run. Truncate (`: > "$LOG_FILE"`) immediately before each test's action when log assertions are involved (T-P3, T-P4, T-P5).
- **G6: T-P3 shim ordering.** The schema seed MUST run with the REAL sqlite3, BEFORE the shim is installed. Either (a) seed in setup() and install shim in the test body, or (b) install a shim with pass-through (more complex). Recommend (a).
- **G7: emit_alert side effects.** `emit_alert` may write to a notification channel. Tests must accept this; if it logs to stderr, suppress with `2>&1 | grep -v alert` or accept the output. Verify by reading `emit_alert` definition before writing tests.
