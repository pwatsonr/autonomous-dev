## 2. Conventions

- Bash files target `bash 5.x`; **every new bash file MUST start with**:
  ```bash
  #!/usr/bin/env bash
  set -euo pipefail
  ```
  and run `shellcheck` clean (no `# shellcheck disable=` without a reason comment).
- All JSON manipulation uses `jq`. No `sed`/`awk` JSON parsing.
- All state.json writes use **temp-file + `mv`** atomicity (TC-06):
  ```bash
  tmp="${state_file}.tmp.$$"
  jq ... "${state_file}" > "${tmp}" && mv "${tmp}" "${state_file}"
  ```
- All timestamps emitted by self-heal code MUST be UTC RFC 3339 with second precision via:
  ```bash
  date -u +"%Y-%m-%dT%H:%M:%SZ"
  ```
- Function naming: `selfheal_*` (public), `_selfheal_*` (private), `detect_<snake>` (detector), `remediate_<snake>` (remediator).
- TypeScript: target the existing `tsconfig.json`; no new tsconfig.
- Event schemas: JSON Schema **draft-07**.
- Tests: bats-core; Jest config already present at `plugins/autonomous-dev/package.json`.

## 3. API Contracts — Bash (`lib/self-heal.sh`)

### 3.1 `selfheal_is_enabled`

```bash
# Returns 0 if AUTONOMOUS_DEV_SELF_HEAL is set to the string "1", else returns 1.
# Never reads any other env var. Never writes.
selfheal_is_enabled() -> 0|1
```

**Postcondition:** Output is the empty string; only the exit code matters.

### 3.2 `selfheal_dispatch`

```bash
# selfheal_dispatch(integration_point, ctx_json) -> 0 | 1 | 2
#
# Parameters:
#   integration_point  one of: review_outcome | session_outcome |
#                              advance_phase | phase_timeout | ledger_check
#   ctx_json           valid JSON string carrying the context object
#                      defined in §6.1 (Context JSON Shape)
#
# Behavior:
#   1. If selfheal_is_enabled fails, return 1 immediately (no-op bypass).
#   2. Look up the ordered list of failure-mode IDs for the integration
#      point (see §3.4).
#   3. For each mode:
#        a. Resolve (detector, event_type, remediator, policy) via
#           _selfheal_table_lookup.
#        b. Call detector with ctx_json via _selfheal_safe_call.
#           - rc=0  -> detected; proceed.
#           - rc=1  -> not detected; continue to next mode.
#           - rc>=2 -> internal error; invoke remediate_capture_and_pause
#                     with mode_id="<mode>" and reason="detector_threw";
#                     return 2.
#        c. Emit the detection event via selfheal_emit_event.
#        d. Call remediator via _selfheal_safe_call.
#           - rc=0  -> emit "<policy>_succeeded" event; record success.
#           - rc=1  -> emit "<policy>_declined" event; continue.
#           - rc>=2 -> invoke remediate_capture_and_pause; return 2.
#   4. After all modes:
#        - rc=0  if at least one remediator succeeded
#        - rc=1  if no detection fired OR no remediator succeeded
#
# Returns:
#   0  caller SHOULD skip the legacy failure path
#   1  caller continues with legacy behavior
#   2  fatal; caller MUST treat the request as paused
selfheal_dispatch() -> 0|1|2
```

### 3.3 `_selfheal_safe_call`

```bash
# _selfheal_safe_call(fn_name, [args...]) -> int
# Invokes "$fn_name" "$@" inside a brace group with `||` rc-capture (NOT a
# subshell) so that a `set -e` failure inside the callee does not unwind
# the supervisor loop AND the callee can still write to the
# _SELFHEAL_LAST_EVIDENCE global (see §14.1).
# Returns the callee's exit code unchanged (capped at 255).
_selfheal_safe_call() -> 0..255
```

```bash
# Pure lookup. Prints to stdout: "<detector>|<event_type>|<remediator>|<policy>"
```

### 4.1 `selfheal_state_set`

```bash
# selfheal_state_set(state_file, dot_path, json_value) -> 0|2
#
# dot_path:     period-separated path RELATIVE to .current_phase_metadata.self_heal
#               e.g. "review_loop.code-reviewer.count" sets
#               .current_phase_metadata.self_heal.review_loop["code-reviewer"].count
# json_value:   a valid JSON literal (string, number, object, etc.) passed
#               through `jq --argjson v`.
#
# Behavior:
#   1. If state_file does not exist OR jq parse fails -> rc 2.
#   2. Write to "${state_file}.tmp.$$" via jq setpath; on success mv to
#      state_file; on failure unlink tmp and return 2.
#   3. Other keys in state.json are byte-preserved (verified by snapshot test).
#
# Returns: 0 success | 2 failure
selfheal_state_set(state_file, dot_path, json_value) -> 0|2
```

**Implementation requirement** — uses this exact jq invocation (or semantically equivalent):

```bash
jq --arg p "$dot_path" --argjson v "$json_value" \
   '.current_phase_metadata //= {}
    | .current_phase_metadata.self_heal //= {}
    | .current_phase_metadata.self_heal |=
        ( . as $cur | $cur | setpath(($p|split(".")); $v) )' \
   "$state_file"
```

### 4.2 `selfheal_state_get`

```bash
selfheal_state_get(state_file, dot_path) -> 0
```

## 7. Detector Specifications

For each detector below: **Inputs** lists required ctx_json keys.
**Returns** 0 (detected), 1 (not detected), or ≥2 (internal error caught
by `_selfheal_safe_call`). On detected, the detector populates
`ctx.evidence` (callers MUST round-trip evidence via the
`_SELFHEAL_LAST_EVIDENCE` global written by the detector; see §14.1 for
the concurrency model that makes this safe).

### 7.1 `detect_review_gate_loop` (F1)

- **Inputs:** `request_id, phase, gate_decision, state_file`
- **Algorithm:**
  1. `outcome = jq '.outcome' gate_decision`. If `!= "REQUEST_CHANGES"` → return 1.
  2. `reviewer = jq '.results[0].reviewer_name // empty'`. If empty → return 1.
  3. `reason = jq '.reason // .results[0].error_message // ""'`.
  4. `fingerprint = printf '%s' "$reason" | head -c 256 | shasum -a 256 | cut -d' ' -f1`.
  5. Read prior `review_loop.<reviewer>` via `selfheal_state_get`.
  6. If `prior.last_reason_fingerprint == fingerprint` → `count = prior.count + 1`; else `count = 1`.
  7. Persist `{count, last_reason_fingerprint: fingerprint, last_seen_at: NOW}` via `selfheal_state_set "review_loop.<reviewer>"`.
  8. If `count >= AUTONOMOUS_DEV_SELF_HEAL_REVIEW_LOOP_THRESHOLD` → set evidence `{reviewer, fingerprint, count, last_seen_at}` → return 0; else return 1.
- **Evidence schema:** `{reviewer: string, fingerprint: string, count: integer, last_seen_at: string}`.

### 7.2 `detect_reviewer_timeout_repeated` (F2)

- **Inputs:** `gate_decision, reviewer_name, state_file`
- **Algorithm:**
  1. `error_kind = jq '.results[] | select(.reviewer_name=="<rev>") | .error_kind // empty'`.
  2. If `error_kind != "reviewer_timeout"` → return 1.
  3. Read `reviewer_timeouts.<reviewer_name>` (default 0); increment.
  4. Persist.
  5. If incremented `>= AUTONOMOUS_DEV_SELF_HEAL_REVIEWER_TIMEOUT_THRESHOLD` → evidence `{reviewer, timeout_count, current_timeout_seconds}` → return 0; else return 1.
- **Evidence schema:** `{reviewer: string, timeout_count: integer, current_timeout_seconds: integer}`.

### 7.3 `detect_phase_timeout_with_progress` (F3)

- **Inputs:** `pre_tree, post_tree, timeout_seconds, elapsed_seconds`
- **Algorithm:**
  1. If `pre_tree` or `post_tree` empty → return 1.
  2. Call existing `working_tree_advanced "$pre_tree" "$post_tree"` (from `supervisor-loop.sh`). If returns 0 (advanced) → set evidence → return 0; else return 1.
- **Evidence schema:** `{pre_tree, post_tree, elapsed_seconds: integer, timeout_seconds: integer}`.
- **MUST be called only on the hard-timeout path** (not soft-timeout); enforced by the integration hook in §11.

### 7.4 `detect_reviewer_error` (F4)

- **Inputs:** `result_file, reviewer_name` (optional)
- **Algorithm:**
  1. `status = jq '.status' result_file`. If `!= "error"` → return 1.
  2. `reason = jq '.status_reason // ""'`.
  3. If `reason ∉ {REVIEW_GATE_CLI_NONZERO, REVIEW_GATE_BAD_JSON, REVIEW_GATE_RESULT_WRITE_FAILED}` → return 1.
  4. `stdout_head = jq '.stdout // ""' | head -c 1024`.
  5. Evidence `{reviewer_name, status_reason: reason, exit_code: jq '.exit_code // null', stdout_head}`. Return 0.
- **Evidence schema:** `{reviewer_name: string | null, status_reason: string, exit_code: integer | null, stdout_head: string}`.

### 7.5 `detect_suspicious_empty` (F5)

- **Inputs:** `result_file, session_log_path, phase`
- **Algorithm:**
  1. If `phase ∉ {prd, tdd, plan, spec, code}` → return 1.
  2. If `jq '.status' result_file != "pass"` → return 1.
  3. `artifact_count = jq '(.artifacts // []) | length'`.
  4. `session_duration = mtime(result_file) - mtime(session_log_path)` (integer seconds; if computation fails → return 1).
  5. If `artifact_count == 0 AND session_duration < MIN_PHASE_DURATION_SECONDS` → set evidence → return 0; else return 1.
- **Evidence schema:** `{phase, artifact_count: integer, session_duration_seconds: integer, threshold: integer}`.

### 7.6 `detect_suspicious_fast` (F6)

- **Inputs:** `result_file, session_log_path, phase` (NOTE: `request_type` may be present in ctx but is INTENTIONALLY UNUSED — see "Baseline keying" below).
- **Algorithm:**
  1. `session_duration = mtime(result_file) - mtime(session_log_path)`.
  2. Read baseline from `${AUTONOMOUS_DEV_STATE_DIR:-$HOME/.autonomous-dev}/self-heal/phase-duration-medians.json`. **Key: `"<phase>"`** (a single string — NOT a compound `phase|request_type` key — per TDD OQ-3 resolution).
  3. If baseline missing/0 → return 1.
  4. If `session_duration < (baseline / SUSPICIOUS_FAST_RATIO) AND session_duration < MIN_PHASE_DURATION_SECONDS` → evidence → return 0; else return 1.
- **Evidence schema:** `{phase, session_duration_seconds: integer, baseline_seconds: integer, ratio: number}`.
- **Baseline keying (locked in by TC-044):** v1 keys baselines by `phase` only. A regression that introduces per-(phase, request_type) keying — even via additive JSON nesting — MUST update TC-044 in the same PR. This is enforced by the dedicated test in §13.

### 7.7 `detect_verification_false_negative` (F7)

- **Inputs:** `result_file, project, phase, phase_started_at`
- **Algorithm:**
  1. If `jq '.status' result_file != "fail"` → return 1.
  2. `reason = jq '.status_reason // ""'`.
  3. Locate test-result artifact, in order:
     - `jq '.test_results_path // .test_artifact // empty' result_file`
     - first match of regex `[A-Za-z0-9_/.\-]+test-results\.json` inside `reason`
     - default `<project>/.autonomous-dev/requests/<request_id>/test-results.json`
  4. If artifact missing → emit `self_verify_skipped_missing_artifact` event (single line via `selfheal_emit_event`), return 1.
  5. If `mtime(artifact) < phase_started_at_unix` → emit `self_verify_skipped_stale_artifact` event, return 1.
  6. `failed_count = jq '(.failures // .stats.failures // ([.tests[]? | select(.status=="failed")] | length))' artifact`.
  7. If `failed_count == 0` → set evidence → return 0; else return 1.
- **Evidence schema:** `{artifact_path: string, artifact_mtime: string, failures: integer, original_status: "fail", original_status_reason: string}`.

### 7.8 `detect_novel_failure` (F8)

- **Inputs:** `result_file, state_file`
- **Algorithm:**
  1. `status = jq '.status // ""' result_file` (suppress jq errors → status="").
  2. If `status ∈ {pass, fail, error, timeout}` AND state.json is parseable → return 1.
  3. Otherwise set evidence and return 0.
- **Evidence schema:** `{observed_status: string, state_parseable: boolean, missing_artifact_paths: string[]}`.

### 7.9 `detect_state_ledger_drift` (F9)

- **Inputs:** `request_id, state_file`
- **Algorithm:**
  1. `state_status = jq -r '.status // ""' state_file`.
  2. `db_status = sqlite3 "$INTAKE_DB" "SELECT status FROM requests WHERE id='<id>'"` (empty if no row).
  3. If both non-empty AND `state_status != db_status` → evidence → return 0; else return 1.
- **Evidence schema:** `{state_status: string, db_status: string}`.

## 8. Remediator Specifications

All remediators **MUST be idempotent per (request, phase)**. Idempotency is
enforced by reading state flags BEFORE acting and bailing with rc=1 if
already done. Return values:

- `0` — remediation succeeded; caller skips the legacy fail path.
- `1` — remediation declined (already applied, or not applicable).
- `≥2` — hard failure; dispatcher falls through to `R_CAPTURE_AND_PAUSE`.

### 8.1 `remediate_fall_back_to_single_reviewer` (F1)

- If `selfheal_state_get review_chain_disabled` == `true` → return 1.
- Atomically set:
  - `review_chain_disabled = true`
  - `review_chain_disabled_at = NOW`
  - `review_chain_disabled_for_phase = <phase>`
- Call `emit_alert "review_chain_disabled_for_request" "<msg>"`.
- Return 0.

### 8.2 `remediate_escalate_reviewer_timeout` (F2)

- If `reviewer_timeout_extended.<reviewer>` == `true` → fall through by invoking `remediate_fall_back_to_single_reviewer ctx_json`; return that exit code.
- Else:
  - `new_timeout = current_timeout_seconds * AUTONOMOUS_DEV_SELF_HEAL_REVIEWER_TIMEOUT_MULTIPLIER`
  - Set `reviewer_timeout_extended.<reviewer> = true` and `reviewer_timeout_overrides.<reviewer> = new_timeout`.
  - Emit `selfheal_emit_event reviewer_timeout_extended ...`.
  - Return 0.

### 8.3 `remediate_extend_phase_budget` (F3)

- If `budget_extended_at != null` → return 1.
- `current = resolve_phase_timeout state_file phase`.
- `new = floor(current * AUTONOMOUS_DEV_SELF_HEAL_BUDGET_EXTENSION_FACTOR)` (use `awk 'BEGIN{printf "%d\n", '"$current"'*'"$factor"'}'`).
- Persist `budget_extended_at = NOW`, `budget_extended_from = current`, `budget_extended_to = new`.
- ALSO update top-level `state.phase_started_at = NOW` (use `jq '.phase_started_at = $now' --arg now ...`; this is NOT a self_heal-namespaced field, so use the existing atomic state-write pattern from `supervisor-loop.sh`, not `selfheal_state_set`).
- Emit `phase_budget_extended`. Return 0.

### 8.4 `remediate_retry_then_exclude` (F4)

- `cur = selfheal_state_get reviewer_retry.<reviewer>` (integer; default 0).
- If `cur == 0`:
  - Set `reviewer_retry.<reviewer> = 1`.
  - Emit `reviewer_retry_scheduled`.
  - Return 0 (caller re-dispatches the gate).
- Else (`cur >= 1`):
  - Resolve `blocking` for `<reviewer>` from the chain config at
    `<project>/intake/reviewers/chains/<request_type>.json` via:
    `jq '.request_types["<rt>"]["<gate>"][] | select(.name=="<rev>") | .blocking'`
  - If lookup fails OR field absent → treat as `blocking=true`, emit `reviewer_blocking_default_assumed`.
  - If `blocking == true` → emit `reviewer_failed_blocking`; return 1.
  - Else (`blocking == false`):
    - Append `<reviewer>` to `excluded_reviewers` (deduped).
    - Emit `reviewer_excluded`.
    - Return 0.

### 8.5 `remediate_requeue_author_phase_once` (F5/F6)

- If `selfheal_state_get requeued.<phase>` == `true` → invoke `remediate_capture_and_pause` with reason `"requeue_exceeded"`; return its rc.
- Else:
  - `mv "$result_file" "${result_file}.suspicious-$(date -u +%Y%m%dT%H%M%SZ).bak"`
  - Atomically (jq via standard state-write pattern, not `selfheal_state_set` because top-level fields are touched):
    - `.status = "running"`
    - `.current_phase_metadata.dispatched_phase = null`
    - `.current_phase_metadata.self_heal.suspicious_previous_result = true`
    - `.current_phase_metadata.self_heal.requeued.<phase> = true`
    - `.current_phase_metadata.self_heal.requeue_reason = "suspicious_<empty|fast>"`
  - **MUST NOT** increment `.retry_count`.
  - Emit `phase_requeued_suspicious`.
  - Return 0.

### 8.6 `remediate_self_verify` (F7)

- If `jq '.self_verified == true' result_file` → return 1 (idempotent).
- If detector evidence reports `failures > 0` → return 1.
- Atomically rewrite `result_file`:
  - `.status = "pass"`
  - `.self_verified = true`
  - `.original_status = "fail"`
  - `.original_status_reason = <prior status_reason>`
  - `.self_verified_at = NOW`
- Emit `verification_false_negative_corrected`.
- Return 0.

### 8.7 `remediate_capture_and_pause` (F8 + universal fallback)

- Compose diagnostic bundle (JSON object):
  ```jsonc
  {
    "schema_version": 1,
    "request_id": "<id>",
    "phase": "<phase>",
    "detected_failure_mode": "<mode_id>",
    "paused_reason_detail": "<short string supplied by caller>",
    "supervisor_version": "<output of: autonomous-dev --version or git rev-parse --short HEAD>",
    "state_json": <full parsed state.json>,
    "last_200_session_log_lines": "<tail -n 200 session_log_path | jq -Rs .>",
    "last_50_events": "<tail -n 50 events.jsonl as JSON array, parse-per-line; on parse-fail include the raw line as string>",
    "non_secret_env": { /* see allow-list below */ },
    "issue_filed_url": null | "https://..."
  }
  ```
- **Env allow-list** (`SELFHEAL_ENV_ALLOWLIST`, declared at top of `self-heal.sh`):
  ```bash
  SELFHEAL_ENV_ALLOWLIST=(
    AUTONOMOUS_DEV_SELF_HEAL
    AUTONOMOUS_DEV_SELF_HEAL_MIN_PHASE_DURATION_SECONDS
    AUTONOMOUS_DEV_SELF_HEAL_REVIEW_LOOP_THRESHOLD
    AUTONOMOUS_DEV_SELF_HEAL_REVIEWER_TIMEOUT_THRESHOLD
    AUTONOMOUS_DEV_SELF_HEAL_BUDGET_EXTENSION_FACTOR
    AUTONOMOUS_DEV_SELF_HEAL_REVIEWER_TIMEOUT_MULTIPLIER
    AUTONOMOUS_DEV_SELF_HEAL_DIAG_BUNDLE_MAX_BYTES
    AUTONOMOUS_DEV_SELF_HEAL_SUSPICIOUS_FAST_RATIO
    AUTONOMOUS_DEV_SELF_HEAL_VALIDATE_SCHEMA
    AUTONOMOUS_DEV_SELF_HEAL_FILE_ISSUES
    AUTONOMOUS_DEV_SELF_HEAL_ISSUE_REPO
    AUTONOMOUS_DEV_STATE_DIR
    AUTONOMOUS_DEV_MAX_SOFT_TIMEOUT_REENTRIES
  )
  ```
- **Denylist regex** (defense-in-depth, applied AFTER allow-list):
  any var name matching `(TOKEN|SECRET|KEY|PASSWORD|CREDENTIAL)` (case-insensitive) is REMOVED even if on the allow-list. Values are NEVER captured for vars matching this regex.
- **Size cap:** total bundle ≤ `AUTONOMOUS_DEV_SELF_HEAL_DIAG_BUNDLE_MAX_BYTES` (default 1 MiB). Order of truncation: (1) drop session-log lines from the head until under cap; (2) if still over, truncate `state_json` value to a placeholder `"<truncated, full file at: <path>>"`.
- **Issue filing** (opportunistic): if `AUTONOMOUS_DEV_SELF_HEAL_FILE_ISSUES=1` AND `gh auth status` exits 0 AND `gh` is in `PATH`:
  - `gh issue create --repo "$AUTONOMOUS_DEV_SELF_HEAL_ISSUE_REPO" --title "Novel failure: <mode_id> in REQ-XXXX" --body-file <bundle-summary>` (the body is a short markdown summary; not the full bundle).
  - On success: set `issue_filed_url = <stdout url>`. On failure: leave `null`.
- Write atomically: `${bundle_tmp} = ...tmp.$$`; `jq -nc "$bundle"` → tmp → `mv`.
- Bundle path: `<project>/.autonomous-dev/requests/<id>/diagnostic-bundle-<UTC-compact-ts>.json` (filename token format: `%Y%m%dT%H%M%SZ`).
- Emit two events in order: `novel_failure_detected`, then `diagnostic_bundle_written`.
- Call existing `escalate_to_paused "$request_id" "$project" "$phase" "$retry_count"` with `paused_reason="novel_failure: <mode_id>"` (NOTE: existing `escalate_to_paused` reads paused_reason from state — the caller MUST `jq` it into `.paused_reason` BEFORE invoking, using the standard state-write pattern).
- Return 0 even on issue-filing failure (issue filing is best-effort).
- Return ≥2 ONLY if the bundle write itself fails irrecoverably; in that case the caller must invoke `escalate_to_paused` directly with `paused_reason="novel_failure_bundle_failed"`.

### 8.8 `remediate_reconcile_ledger` (F9)

- Call existing `sync_intake_db_row "$request_id" "$current_phase" "$state_status" "$(date -u +%Y-%m-%dT%H:%M:%SZ)"`.
- If rc != 0 → return 2.
- Persist `ledger_reconciled_at = NOW`.
- Emit `ledger_reconciled`.
- Return 0.

```ts
/**
 * Options for `runReviewers`. All optional.
 */
export interface RunReviewersOpts {
  /** Reviewer names to skip entirely (never invoked). */
  excludedReviewers?: string[];
  /**
   * If true, when a reviewer returns verdict === 'ERROR' the runner
   * re-invokes that reviewer ONCE per gate. If the retry also errors,
   * the second ERROR result is the one returned.
   */
  retryOnce?: boolean;
}

/**
 * Run all reviewers in `chain` against `ctx`, honoring per-reviewer
 * exclusion and one-shot retry. Returns one ReviewerResult per
 * non-excluded reviewer, in chain order.
 *
 * Behavior:
 *   - For each entry in chain:
 *       - if entry.name ∈ excludedReviewers: skip (no result emitted)
 *       - else invoke via deps.invoke
 *       - if result.verdict === 'ERROR' AND opts.retryOnce: invoke once
 *         more; return whichever result we end with (first if non-error,
 *         second on retry).
 *
 * The function is a pass-through to the existing `ReviewerRunner` for
 * concurrency / scheduling; this wrapper adds only the exclusion +
 * retry policy.
 */
export async function runReviewers(
  chain: ReviewerEntry[],
  ctx: ChangeSetContext,
  opts: RunReviewersOpts,
  deps: { invoke: InvokeReviewerFn; emit?: TelemetryEmitFn },
): Promise<ReviewerResult[]>;
```

## 12. Error Handling

| Error condition | Where | Handling |
|---|---|---|
| `_selfheal_safe_call` callee returns ≥2 | dispatcher | Invoke `remediate_capture_and_pause`; dispatcher returns 2 |
| `selfheal_state_set` jq parse failure | state-writer | Return 2 → propagates to ≥2 in remediator → triggers capture+pause |
| Schema validation failure | event emitter | `log_warn`; event STILL appended (FR-DETECT-05 / ADR-4) |
| Diagnostic bundle write failure | `remediate_capture_and_pause` | `log_error`; call existing `escalate_to_paused` directly with `paused_reason="novel_failure_bundle_failed"`; return 2 |
| `gh` CLI absent / unauthenticated / fails | `remediate_capture_and_pause` | Set `issue_filed_url = null`; do NOT raise; return 0 |
| Chain-config lookup missing `blocking` | `remediate_retry_then_exclude` | Treat as `blocking=true`; emit `reviewer_blocking_default_assumed` event |
| `AUTONOMOUS_DEV_SELF_HEAL=0` | dispatcher | All `selfheal_dispatch` calls return 1; no events emitted; legacy paths run unchanged |
| `flock` absent (macOS dev) | event emitter | Fall back to `set -C` lockfile pattern; document in module header |
| ajv absent (CI without dev deps) | event emitter | Skip schema validation; emit a one-time `log_warn`; event still appended |

- `"novel_failure: F8 (see diagnostic-bundle-<ts>.json)"`

- `"reviewer_chain_disabled_and_single_reviewer_still_blocked"`

## 14. Implementation Notes

### 14.1 Bash idioms

- Use `mapfile -t` (bash 4+) for collecting jq arrays into bash arrays.
- For atomic top-level state writes (NOT self-heal-scoped fields), follow the existing pattern in `supervisor-loop.sh`:
  ```bash
  local tmp="${state_file}.tmp.$$"
  jq '<filter>' "$state_file" > "$tmp" && mv "$tmp" "$state_file"
  ```
- **`_SELFHEAL_LAST_EVIDENCE` global — concurrency model (READ THIS):**
  Detectors surface their `evidence` JSON to the dispatcher via the
  process-global `_SELFHEAL_LAST_EVIDENCE`. This is **safe by
  construction** under the supervisor's concurrency model:
  - The autonomous-dev supervisor loop (`supervisor-loop.sh`) is
    **strictly single-threaded** — it processes one request × one
    phase × one dispatch at a time. `selfheal_dispatch` is therefore
    invoked sequentially, and only one `detect_*` / `remediate_*` pair
    is in flight in the bash process at any moment.
  - `_selfheal_safe_call` (§3.3) uses a brace group `{ ; }` with
    `|| rc=$?` rc-capture rather than a `( )` subshell precisely so
    the callee's assignment to `_SELFHEAL_LAST_EVIDENCE` is visible
    to the dispatcher in the same shell. Switching to a subshell
    would silently break evidence round-trip.
  - **Future-misuse warning:** any future change that introduces
    `xargs -P`, `&`-backgrounding, parallel `for ... &` over multiple
    requests, or any code path that forks the supervisor into
    concurrent dispatchers in the SAME bash process will break this
    invariant and MUST replace the global with a stdout-passed JSON
    string from detector to dispatcher. The module header of
    `self-heal.sh` MUST include a short comment block stating this
    invariant verbatim so a future maintainer cannot miss it.
  ```bash
  _SELFHEAL_LAST_EVIDENCE='{"reviewer":"X","fingerprint":"...","count":3,"last_seen_at":"..."}'
  return 0
  ```
  Pattern for `_selfheal_safe_call`:
  ```bash
  _selfheal_safe_call() {
      local fn="$1"; shift
      local rc=0
      _SELFHEAL_LAST_EVIDENCE='{}'
      { "$fn" "$@"; rc=$?; } || rc=$?
      return "$rc"
  }
  ```
  (Trap-protected brace group, NOT a `( )` subshell, so evidence assignment persists.)

### 14.2 macOS portability

- `flock` is not installed by default on macOS. Detect with `command -v flock`; fall back to:
  ```bash
  (
    set -C    # noclobber
    until : > "${file}.lock" 2>/dev/null; do sleep 0.05; done
    trap 'rm -f "${file}.lock"' EXIT
    printf '%s\n' "$line" >> "$file"
  )
  ```
- `shasum -a 256` works on both macOS and Linux; do NOT use `sha256sum` (Linux-only).
- `date -u` flags are identical; OK to use.
