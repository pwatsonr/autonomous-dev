## API Contracts

Two new internal bash functions are added to `supervisor-loop.sh`. No external/CLI surface changes. Place both **above `advance_phase()`** (i.e. before `:2232`) and **below the `log_*` helpers** so the logging functions and `$INTAKE_DB` global are defined.

### Contract 1 — `map_state_status_to_intake()`

```bash
# map_state_status_to_intake(state_status) -> echoes mapped intake status; rc 0/1
#   Pure function. No side effects. No I/O. No logging.
#   queued        -> "queued"
#   running|gate  -> "active"
#   paused        -> "paused"
#   done          -> "done"
#   failed        -> "failed"
#   <anything>    -> echo "" ; return 1
map_state_status_to_intake() {
    local s="$1"
    case "$s" in
        queued)        echo "queued" ;;
        running|gate)  echo "active" ;;
        paused)        echo "paused" ;;
        done)          echo "done"   ;;
        failed)        echo "failed" ;;
        *)             echo ""; return 1 ;;
    esac
    return 0
}
```

| Aspect | Spec |
|--------|------|
| Param `$1` | `state_status` — a `state.json` status string. Required. No default. |
| stdout | The mapped intake-vocabulary token, or `""` (empty) when unmapped. |
| Return code | `0` when mapped; `1` when unmapped. |
| Preconditions | None. |
| Postconditions | Exactly one of: (mapped token + rc 0) OR (empty stdout + rc 1). Every emitted non-empty token ∈ `{queued, active, paused, done, failed}` ⊂ the `requests.status` CHECK set. |
| Side effects | None (no logging, no file/db access). |
| Invariants | `cancelled` is **never** emitted — cancellation is owned by orphan/cancel paths only (TDD NG4). |

**Usage example**
```bash
map_state_status_to_intake running   # stdout "active",  rc 0
map_state_status_to_intake gate       # stdout "active",  rc 0
map_state_status_to_intake bogus      # stdout "",        rc 1
```

### Contract 2 — `sync_intake_db_row()`

```bash
# sync_intake_db_row(request_id, phase, state_status, ts) -> always returns 0
#   Best-effort, non-fatal mirror of a state.json transition into intake.db.
#   Logs exactly one terminal line on every path. Never aborts the caller.
sync_intake_db_row() {
    local request_id="$1" phase="$2" state_status="$3" ts="$4"

    # 1) sqlite3 must be available.
    if ! command -v sqlite3 >/dev/null 2>&1; then
        log_error "sync_intake_db_row: sqlite3 CLI not found; cannot mirror ${request_id} -> ${phase}"
        return 0
    fi

    # 2) DB file must exist.
    if [[ ! -f "$INTAKE_DB" ]]; then
        log_warn "sync_intake_db_row: ${INTAKE_DB} not present; skipping ledger update for ${request_id}"
        return 0
    fi

    # 3) Map status; skip on unmapped/empty.
    local mapped
    if ! mapped=$(map_state_status_to_intake "$state_status") || [[ -z "$mapped" ]]; then
        log_error "sync_intake_db_row: unmapped state status '${state_status}' for ${request_id}; not writing ledger"
        return 0
    fi

    # 4) Parameter-bound UPDATE + changes() in one connection.
    local rid="${request_id//\'/\'\'}"
    local ph="${phase//\'/\'\'}"
    local update_out rc
    update_out=$(sqlite3 "$INTAKE_DB" <<SQL 2>&1
.parameter set :rid '${rid}'
.parameter set :phase '${ph}'
.parameter set :status '${mapped}'
.parameter set :ts '${ts}'
UPDATE requests
   SET current_phase = :phase,
       status        = :status,
       updated_at    = :ts
 WHERE request_id = :rid;
SELECT changes();
SQL
)
    rc=$?

    # 5) Non-zero exit -> ERROR, non-fatal.
    if [[ $rc -ne 0 ]]; then
        log_error "sync_intake_db_row: sqlite3 UPDATE failed for ${request_id} -> (${phase}, ${mapped}); rc=${rc}; out=${update_out}"
        return 0
    fi

    # 6) Zero-row match -> WARN.
    local changed
    changed=$(printf '%s\n' "$update_out" | tail -n1)
    if [[ "$changed" == "0" ]]; then
        log_warn "sync_intake_db_row: 0 rows updated for ${request_id} (no ledger row?) -> (${phase}, ${mapped})"
        return 0
    fi

    # 7) Success.
    log_info "sync_intake_db_row: ledger updated ${request_id} -> (${phase}, ${mapped})"
    return 0
}
```

| Aspect | Spec |
|--------|------|
| Param `$1` `request_id` | Request ID (e.g. `REQ-000013`). Required. Single-quote-escaped before binding. |
| Param `$2` `phase` | Phase name written to `current_phase` verbatim (free-form TEXT). Required. Single-quote-escaped. |
| Param `$3` `state_status` | A `state.json` status; mapped via Contract 1. Required. |
| Param `$4` `ts` | ISO-8601 UTC timestamp (`%Y-%m-%dT%H:%M:%SZ`). Required. Written to `updated_at` verbatim. |
| stdout | None meaningful (all signalling via `log_*`). |
| Return code | **Always `0`** — callers must never branch on it (enforces the non-fatal contract). |
| Preconditions | `$INTAKE_DB` global set; `log_*` defined; `map_state_status_to_intake` defined. |
| Postconditions (happy) | The `requests` row with `request_id=$1` has `current_phase=$2`, `status=map($3)`, `updated_at=$4`; INFO logged. |
| Side effects | One `sqlite3` process spawn; one bounded `UPDATE` against `$INTAKE_DB`; one `log_*` line. |
| Invariants | Exactly one terminal log line per invocation (no silent return). `state.json` is never touched. No `INSERT` is ever issued (0-row → WARN only). |

**Error conditions** (all non-fatal, rc 0):

| Trigger | Log level | Message shape |
|---------|-----------|---------------|
| `sqlite3` not on PATH | ERROR | `…sqlite3 CLI not found; cannot mirror <rid> -> <phase>` |
| `$INTAKE_DB` file missing | WARN | `…<db> not present; skipping ledger update for <rid>` |
| `map_*` returns 1 / empty | ERROR | `…unmapped state status '<status>' for <rid>; not writing ledger` |
| `sqlite3` non-zero exit | ERROR | `…sqlite3 UPDATE failed for <rid> -> (<phase>, <mapped>); rc=<rc>; out=<text>` |
| `changes()==0` | WARN | `…0 rows updated for <rid> (no ledger row?) -> (<phase>, <mapped>)` |
| `changes()>0` | INFO | `…ledger updated <rid> -> (<phase>, <mapped>)` |

**Usage example**
```bash
sync_intake_db_row "REQ-000013" "prd" "running" "2026-05-20T00:00:00Z"
# -> requests row: current_phase='prd', status='active', updated_at='2026-05-20T00:00:00Z'; INFO logged
```

## Error Handling

### Error taxonomy for this spec
All failures are **operator-facing daemon log lines**, never user-facing, never fatal. There are no exceptions/exit codes propagated to callers — `sync_intake_db_row` always returns `0`.

| Category | Recoverable? | Strategy |
|----------|--------------|----------|
| `sqlite3` missing on PATH | Fatal-to-ledger, recoverable-to-daemon | log ERROR, skip write, continue advance |
| `intake.db` file absent | recoverable | log WARN, skip write, continue advance |
| Unmapped `state_status` | recoverable | log ERROR, skip write, continue advance |
| `sqlite3` non-zero exit (locked / read-only / corrupt) | recoverable | log ERROR with captured `out`, continue advance |
| 0 rows updated (no ledger row) | recoverable | log WARN, **no INSERT**, continue advance |

### Recovery principle
`state.json` is authoritative and is written **before** the mirror. A ledger-write failure leaves `state.json` correct; the next phase boundary will re-attempt the mirror with fresh values. The original bug class (silent miss) is eliminated because every path logs exactly once (TDD NFR-6 / SM-6).

## Implementation Notes

- **IN-1 (critical — `INTAKE_DB` is `readonly`).** `readonly INTAKE_DB="${DAEMON_HOME}/intake.db"` (`:18`) and `DAEMON_HOME="${HOME}/.autonomous-dev"` (`:17`) are evaluated at source time. Tests **cannot** reassign `INTAKE_DB` after sourcing (the TDD §8 snippet `INTAKE_DB="$TEST_WORK_DIR/intake.db"` would error with `readonly variable`). **Required harness pattern** in each bats `setup()`: set `HOME` to a temp dir **before** sourcing the script, so `INTAKE_DB` resolves under the temp tree. Example:
  ```bash
  setup() {
      PLUGIN_DIR="$(cd "${BATS_TEST_DIRNAME}/../.." && pwd)"
      TEST_WORK_DIR="$(mktemp -d)"
      export HOME="$TEST_WORK_DIR"          # <-- BEFORE source; makes DAEMON_HOME/INTAKE_DB temp
      export LOG_FILE="$TEST_WORK_DIR/daemon.log"
      set +e; source "$PLUGIN_DIR/bin/supervisor-loop.sh"; set -e
      TEST_PROJECT="$TEST_WORK_DIR/test-project"
      TEST_REQUEST_ID="REQ-260512"
      TEST_REQ_DIR="$TEST_PROJECT/.autonomous-dev/requests/$TEST_REQUEST_ID"
      mkdir -p "$TEST_REQ_DIR"
  }
  ```
  Note this differs from the existing `advance_phase.bats:5–23` ordering (which sources before `mktemp`). The new/updated tests must source **after** `HOME` is set. Confirm `$LOG_FILE` is also writable in the temp tree so `log_*` assertions can `grep` it.
- **IN-2 (escaping/heredoc parity).** Copy the orphan-path idiom exactly: `${var//\'/\'\'}` for `request_id` and `phase`, `.parameter set :name 'value'`, single `sqlite3 "$INTAKE_DB" <<SQL … SQL` heredoc. Reference: `:2778–2787`.
