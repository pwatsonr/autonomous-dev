# Phase 15 — Specialist reviewer chains

This phase configures specialist reviewer chains (security,
performance, accessibility, db-migration, dependency-update, etc.)
that run during code review. The phase enumerates the bundled catalog
data-driven from `specialist-reviewers.json`, lets the operator
enable/disable each specialist with chain-order weight + threshold,
writes `<repo>/.autonomous-dev/reviewer-chains.yaml`, and verifies
via `autonomous-dev reviewer-chain dry-run`.

The dry-run is **strictly read-only**: 0 LLM invocations, 0
side-effects beyond reading the chain YAML.

## Steps

### Step `read-catalog`

```bash
catalog="$PLUGIN_DIR/config/specialist-reviewers.json"
if [[ ! -f "$catalog" ]]; then
  echo "[phase-15] specialist-reviewers catalog missing: $catalog" >&2
  echo "  See: /autonomous-dev-assist:troubleshoot" >&2
  exit 1
fi
jq -e 'type == "array"' "$catalog" >/dev/null \
  || { echo "[phase-15] catalog malformed (not a JSON array)" >&2; exit 1; }
```

### Step `existing-decision`

```bash
target="$REPO/.autonomous-dev/reviewer-chains.yaml"
if [[ -f "$target" ]]; then
  read -r -p "reviewer-chains.yaml exists. (k)eep / (r)eplace? [k] " choice
  case "${choice:-k}" in
    r|R) EXISTING_DECISION=replace ;;
    *)   EXISTING_DECISION=keep ;;
  esac
fi
```

Note: merge is intentionally NOT supported (chain order is operator-
determined; conflict resolution is non-trivial).

### Step `enumerate`

```bash
declare -a ENABLED_IDS
declare -A WEIGHTS
declare -A THRESHOLDS
while IFS= read -r entry; do
  id="$(jq -r '.id' <<<"$entry")"
  desc="$(jq -r '.description // ""' <<<"$entry")"
  default_w="$(jq -r '.default_weight // 100' <<<"$entry")"
  default_t="$(jq -r '.default_threshold // 0.7' <<<"$entry")"
  read -r -p "Enable specialist '$id' ($desc)? [y/N] " ans
  case "$ans" in
    y|Y)
      tries=0
      while :; do
        read -r -p "  weight (lower=earlier in chain) [default $default_w]: " w
        w="${w:-$default_w}"
        if [[ "$w" =~ ^[0-9]+$ ]]; then break; fi
        tries=$((tries+1))
        echo "  weight must be numeric" >&2
        if (( tries >= 3 )); then
          echo "[phase-15] non-numeric weight after 3 attempts; aborting" >&2
          exit 1
        fi
      done
      read -r -p "  threshold [default $default_t]: " t
      t="${t:-$default_t}"
      ENABLED_IDS+=("$id")
      WEIGHTS[$id]="$w"
      THRESHOLDS[$id]="$t"
      ;;
    *) ;;
  esac
done < <(jq -c '.[]' "$catalog")
```

### Step `sort-and-render`

Sort enabled specialists by weight ascending, ties broken by id
ascending. Write to a temp file, then atomic-rename to the live path
(FR-14).

```bash
tmp="${target}.tmp"
{
  echo "# Auto-generated by setup-wizard phase 15. Do not edit by hand."
  echo "specialists:"
  # Sort ascending by weight then id
  for id in $(printf '%s\n' "${ENABLED_IDS[@]}" \
              | awk -v w="$(declare -p WEIGHTS)" 'BEGIN{ }{ print $0 }' \
              | sort -t'|' -k2,2n -k1,1); do
    printf '  - id: %s\n    weight: %s\n    threshold: %s\n' \
      "$id" "${WEIGHTS[$id]}" "${THRESHOLDS[$id]}"
  done
} > "$tmp"
mv "$tmp" "$target"
```

(Bash sort with associative-array keys is finicky; the implementation
may pre-emit `id|weight` lines, sort, then re-emit. Documented for
clarity; tests verify byte-identical output across replays per
FR-9.)

### Step `dry-run-verify`

```bash
out="$(autonomous-dev reviewer-chain dry-run --against HEAD~1..HEAD --json)" \
  || { echo "[phase-15] dry-run failed" >&2; exit 1; }
# Each enabled specialist must appear in the dispatched_specialists list.
for id in "${ENABLED_IDS[@]}"; do
  jq -e --arg id "$id" '.dispatched_specialists | map(.id) | index($id) != null' <<<"$out" \
    >/dev/null \
    || { echo "[phase-15] specialist $id not dispatched" >&2; exit 1; }
done
```

### Step `write-config`

```bash

   --arg ts "$(date -u +%Y-%m-%dT%H:%M:%SZ)" \
   --argjson n "${#ENABLED_IDS[@]}" \

    | .reviewer_chains.last_dry_run_at=$ts
    | .reviewer_chains.specialists_enabled_count=$n' \

```

## Defense-in-depth

- The cost-counter check guards against LLM invocations during dry-
  run; FR-16 / SPEC-021-3-02 contract this as zero.
- Atomic-rename (`reviewer-chains.yaml.tmp` → `reviewer-chains.yaml`)
  prevents corrupted half-written files on Ctrl-C.
- Merge is not supported; the operator chooses keep/replace only.
- Chain ordering is deterministic: weight asc, ties broken by id
  asc. Same operator inputs produce byte-identical output.

## Resume contract

`WIZARD_RESUME_STEP` jumps to a named step. Mid-enumerate SIGTERM
preserves the wizard checkpoint at the most recently completed
specialist; resume continues at the next un-prompted catalog entry.
Any orphan `.tmp` file is unlinked at the start of `sort-and-render`.
