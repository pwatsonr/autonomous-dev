## Description
Implement the test suite for cost aggregation, SVG chart rendering, log tailing, redaction, ring-buffer backpressure, gzip download, and the route handlers. Tests are organized into unit, integration, and end-to-end tiers and use deterministic fixtures (no `Date.now`, no `setTimeout`-based waits). Snapshot tests pin SVG byte-stability for charts. Backpressure tests verify the ring-buffer eviction contract under sustained producer load. Coverage gates ensure the cost/charts/logs surfaces meet the plan's >90% line coverage target.

## Implementation Details

### Fixtures

**`tests/fixtures/cost-ledger-sample.jsonl`**: 30 days of synthetic data, generated by a deterministic builder. Layout:
- 5 distinct repositories
- All 7 phases present each day (some days have only Code+Review entries to test phase zero-fill)
- 200 entries per day for 30 days (= 6000 entries total)
- Cost values seeded from `sin(dayIndex * 0.4)` so they have realistic variance without randomness
- One day with zero entries (day 14) to verify continuous daily zero-fill

**`tests/fixtures/daemon-log-sample.jsonl`**: 1000 lines covering:
- 250 ERROR, 250 WARN, 250 INFO, 250 DEBUG entries
- 100 entries with `request_id` populated; 900 without
- Timestamps spanning 48h to support 1h/4h/24h window testing
- One entry per redaction pattern (api keys, github tokens, AWS keys, JWTs, password fields, emails) — embedded in `message` and `context`
- Three malformed lines (lines 100, 500, 950) for malformed-recovery testing

**`portal/__tests__/helpers/tmp-portal.ts`**:

export interface TmpPortalHandle {

### Unit Tests — `aggregator.test.ts` (12 cases)
1. Empty ledger → all rollups return zero/empty arrays.
2. `loadLedger` skips malformed lines and increments a warn counter.
3. `daily` produces 30 entries for a 30-day window with one zero-fill day.
4. `monthly` aggregates calendar months correctly across a year boundary (Dec 2025 + Jan 2026).
5. `byRepository` percentages sum to 100.00 (±0.01).
6. `byPhase` always returns 7 entries in canonical order, including zero-filled phases.
7. `topExpensive(entries, 10)` returns 10 results sorted by cost descending; ties broken by `request_id` ascending.
8. `topExpensive(entries, 0)` returns `[]`.
9. `projectSevenDay` with full week of data: average over 7 days; projection = avg * 7.
10. `projectSevenDay` with 3 days of data: `basis_days === 3`; projection = 3-day-avg * 7.
11. `projectSevenDay` with no data: returns all zeros, `basis_days === 0`.
12. Negative `cost_usd` entries are dropped during validation.

### Unit Tests — `cap_monitor.test.ts` (8 cases)
1. Daily cap unset → returns `null`.
2. Daily 50% spend → `severity === 'ok'`.
3. Daily 80% spend → `severity === 'warn'`.
4. Daily 100% spend → `severity === 'exceeded'`.
5. Daily 150% spend → `severity === 'exceeded'`, `pct_of_limit === 150`.
6. Monthly cap with day_of_month=10 in a 30-day month, $100 spent → `projected_total_usd === 300.00`.
7. Monthly cap on day 1 → projection = current * 30.
8. Cap config returning negative `daily_usd` is treated as unset.

### Unit Tests — Log Surface

**`parser.test.ts`** (8 cases):
1. Valid NDJSON line → fully populated `LogEntry`.
2. Malformed JSON → `null`, no throw.
3. Missing `timestamp` → `null`.
4. Lowercase level (`"info"`) → coerced to `"INFO"`.
5. Invalid level (`"VERBOSE"`) → `null`.
6. `parseFile(fixture, 500)` returns the most recent 500 entries in chronological order.
7. `parseFile` over the 1000-line fixture in <300ms.
8. `streamFile` yields valid entries one at a time; total count matches `parseFile` (excluding malformed).

**`filter.test.ts`** (10 cases):
1. No criteria → all entries returned.
2. `level=ERROR` → only ERROR entries.
3. `request_id=REQ-000005` → only matching entries.
4. `time_range=1h` excludes entries older than 1h relative to fakeClock.
5. AND combination: `level=ERROR + request_id=REQ-000005` → both must match.
6. Invalid level in query (`level=BOGUS`) is silently dropped → behaves as no level filter.
7. Invalid request_id pattern (`abc`) is dropped.
8. `start_time + end_time` overrides `time_range`.
9. URL round-trip: `toQuery(fromQuery(q)) === q` for every supported field.
10. `apply` over 10,000 entries with all 3 filters set in <500ms.

**`redact.test.ts`** (8 cases): one per pattern (api key, github token, AWS key, JWT, password field, email) plus:
7. Nested `context.headers.authorization` redacted.
8. Original entry not mutated (deep-equal pre/post).

**`ring_buffer.test.ts`** (8 cases):
1. Empty buffer `size().bytes === 0`.
2. Pushing entries below 1MB does not evict.
3. Pushing an entry that exceeds the budget evicts the oldest.
4. After eviction, `size().bytes <= maxBytes`.
5. `takeSince(undefined)` returns full snapshot.
6. `takeSince(timestamp)` returns only newer entries.
7. FIFO order: first inserted is first evicted.
8. Repeated push/evict cycle for 10,000 entries does not leak memory (asserted via `process.memoryUsage().heapUsed` comparison after explicit `gc()` if `--expose-gc` is available; otherwise asserted by buffer count only).

## Notes

- Snapshot tests lock SVG byte-stability; intentional chart changes require a dedicated snapshot-update commit so reviewers see the diff.
- Backpressure tests are flake-prone — use a fake clock plus synchronous push loops (no `await sleep`); flush microtasks via `await Promise.resolve()` between batches.
- The deterministic ledger generator lives at `tests/fixtures/build-cost-ledger.ts`. Commit both the generator and the generated `.jsonl` so CI never regenerates fixtures.
- Redaction tests must include the nested `context.headers.authorization` case — that's where secrets actually appear in production logs.
- Snapshot files live under `tests/fixtures/charts/__snapshots__/` so they regenerate independently of test discovery.
- The E2E scenario is intentionally minimal; deeper UX coverage is deferred to a future plan.
