## Summary
Apply the case-aware token substitutions `Vitest` → `Jest` and `vitest` →
`jest` (both word-boundary-anchored) across every SPEC under
`plugins/autonomous-dev/docs/specs/`, then revert any historical-context
carve-out SPECs identified by SPEC-031-2-01 to their pre-sweep content.
This spec performs the mechanical edits; matrix population and commit are
deferred to SPEC-031-2-04.

## Functional Requirements

- **FR-1**: A find+sed pipeline MUST apply two case-aware substitutions per
  TDD §5.2:
  - `s/\bVitest\b/Jest/g` (preserves capitalization)
  - `s/\bvitest\b/jest/g`
  Both substitutions use word-boundary anchors (`\b`). Task: PLAN-031-2 task 3.
- **FR-2**: All `*.md.bak` files generated by the BSD-compatible
  `sed -i.bak` invocation MUST be deleted before the spec is considered
  complete.
- **FR-3**: For each SPEC on SPEC-031-2-01's carve-out list, the changes
  applied by FR-1 MUST be reverted with `git checkout -- <path>`. The
  hand-amendment for those SPECs is performed by SPEC-031-2-04, not here.
- **FR-4**: After FR-3's reverts, `grep -rliE "\bvitest\b" plugins/autonomous-dev/docs/specs/`
  MUST return either zero matches OR matches limited to the declared
  carve-out file list.
- **FR-5**: `git diff --stat plugins/autonomous-dev/docs/specs/` after
  this spec MUST show modified-file count equal to (audit count) −
  (carve-out count).

## Patterns to Find/Replace

**Find pattern 1 (capitalized)**: `\bVitest\b` (regex; word-boundary-anchored)
**Replace with**: `Jest`

**Find pattern 2 (lowercase)**: `\bvitest\b` (regex; word-boundary-anchored)
**Replace with**: `jest`

**Sed expression**:
```
-e 's/\bVitest\b/Jest/g' -e 's/\bvitest\b/jest/g'
```

The two expressions chain via repeated `-e`; they are NOT combined into
a single substitution because the case mapping must be exact (`Vitest` →
`Jest`, not `Vitest` → `jest`).

## Implementation Notes

- Word-boundary anchors (`\b`) are critical. Without them, `vitestz` would
  become `jestz` (wrong). With them, only standalone tokens match.
- BSD `sed` on macOS supports `\b` only via the `-E` flag (extended regex)
  on some versions. Test the command on macOS before relying on it. If
  `\b` is not honored, fall back to explicit boundary classes:
  `s/\([^[:alnum:]_]\)Vitest\([^[:alnum:]_]\)/\1Jest\2/g` (and equivalent
  for lowercase). Document the fallback in the matrix preamble if used.
- The carve-out reverts MUST happen AFTER the sweep, not before. Selective
  exclusion via `find -not -path` is fragile when the carve-out list grows.
  Sweep-then-revert is more robust.
- Vitest-specific API references (`vi.fn()`, `vi.mock()`, `vi.spyOn()`)
  may remain in some SPECs after this spec. They are in scope for
  SPEC-031-2-03 (the `\bvi\.` review pass), not this spec.
