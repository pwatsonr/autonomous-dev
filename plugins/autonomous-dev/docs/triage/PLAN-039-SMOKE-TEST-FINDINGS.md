| **B-17** | **HIGH** | daemon (agent permissions + synthesis) | **Doc-author/reviewer phases produce zero on-disk artifacts; reviewer-fail verdicts are silently converted to pass.** Two compounding bugs: (1) The doc-author agents (`prd-author`, `tdd-author`, `plan-author`, `spec-author`) and reviewer agents (`doc-reviewer`, `quality-reviewer`, etc.) have frontmatter `tools: Read, Glob, Grep, WebSearch, WebFetch` — **no Write**. So they cannot create artifact files (`docs/prd/<slug>.md`, etc.) and cannot write their own `phase-result-<phase>.json`. They complete narratively (text-only output describing what they *would* have written) and exit 0. (2) The daemon's synthesis fallback (`synthesize_phase_result`, run when no `phase-result-*.json` exists) sets `status: pass` solely from exit-code-0, **even when the agent's text output explicitly returned a BLOCK / status:fail verdict.** Repro on REQ-000008 (2026-05-17, live launchd daemon, plugin 0.1.0 cache): `prd-author` produced narrative text saying "I notice that I don't have a Write tool available to actually create the files" (session-1778981392.txt, $0.29, 16 turns) → no `docs/prd/*.md` exists. `doc-reviewer` for prd_review then said verbatim: `{"status": "fail", "phase": "prd_review", "feedback": "BLOCK - No PRD document exists to review."}` (session-1778981577.txt, $0.14, 15 turns) — and the daemon wrote `{"status":"pass", "synthesized": true, "feedback": "synthesized from exit code 0"}`. Phase advanced. Same pattern on tdd + plan + spec + all four reviews. **Important narrowing: the code/integration/deploy agents DO have Write** — they produce rich agent-written phase-result-*.json files and create real files on disk (`todo.py`, `test_todo.py`, `integration_test.py`, `Dockerfile`, `docker-compose.yml`, `pyproject.toml`, `install.sh`, `DEPLOYMENT.md`, `dist/*.whl`, etc.). It's only the **doc-author phases (prd/tdd/plan/spec) and all reviewer phases (*_review)** that are Read-only by design and never produce on-disk artifacts. The pipeline reaches `monitor/done` and ships real code, just with **no design documents**. Phase 1 acceptance partial-fails on `docs/prd|tdd|plans|specs/<slug>.md` (missing) but passes on `todo/*.py` + `tests/*.py`. **Gate-approval flow (Phase 4) and retry-exhaustion (Phase 8) both can't be tested** because reviewer fails are masked as passes. | Two-part fix: (a) **Doc-author agents need Write tool** — add `Write` to the `tools:` frontmatter in `agents/{prd,tdd,plan,spec}-author.md`. Or: introduce a write-mediator where the daemon parses the agent's output for `## File: <path>\n\`\`\`\n<content>\n\`\`\`` blocks and writes them post-session. (b) **Synthesis must not mask explicit fails** — parse the agent's final result text for a JSON code-block containing `"status": "fail"`/`"block"` and honor it. Test plan: deliberately blocked reviewer must terminate the pipeline in `failed` state. Found 2026-05-17 Phase 1 of full E2E. |
